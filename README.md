# geo-fix — Подмена геолокации для доступа к Google-сервисам

## Что это

Программа для Windows, которая подменяет все сигналы геолокации в браузере, чтобы в связке с VPN обеспечить доступ к NotebookLM, Gemini и другим гео-заблокированным сервисам.

## Что маскируется

| Сигнал | Что подменяется | Где работает |
|--------|----------------|--------------|
| **Часовой пояс** | `Date.getTimezoneOffset()`, `Intl.DateTimeFormat`, `Temporal.Now` | Все сайты |
| **Язык** | `navigator.language/languages`, заголовок `Accept-Language` | Все сайты и запросы |
| **GPS-координаты** | `navigator.geolocation`, Google Geolocation API | Все сайты |
| **WebRTC** | `RTCPeerConnection` переведён в relay-режим — IP не утекает | Все сайты |
| **Разрешения** | `navigator.permissions.query` для геолокации возвращает `granted` | Все сайты |
| **WiFi-геолокация** | Windows Location Services отключается на время работы | Система |

Все подмены работают на **всех сайтах**, не только на Google. Переопределённые функции отвечают `[native code]` при проверке — стандартные антидетект-проверки не обнаруживают подмену.

## Что программа НЕ делает

- НЕ записывает и НЕ хранит ваш интернет-трафик
- НЕ отправляет данные куда-либо
- НЕ меняет системные настройки навсегда — всё возвращается после выключения
- НЕ заменяет VPN — работает только вместе с ним

## Установка

### Готовый архив (рекомендуется)
1. Скачайте `geo-fix-windows.zip` из [Releases](../../releases/latest)
2. Распакуйте в любую папку (например, `C:\geo-fix\`)
3. Запустите `Включить (US).bat`
4. При первом запуске откроется мастер настройки

### Автоустановщик
1. Скачайте весь проект
2. Запустите `install.bat` — он скачает Python, установит зависимости и создаст ярлыки

## Использование

### Ярлыки на рабочем столе

| Ярлык | Действие |
|-------|----------|
| **Включить (US)** | Маскировка под США |
| **Включить (DE)** | Маскировка под Германию |
| **Включить (NL)** | Маскировка под Нидерланды |
| **Включить (GB)** | Маскировка под Великобританию |
| **Выключить** | Остановить, вернуть всё как было |
| **Починить интернет** | Восстановить настройки после сбоя |

### Командная строка

```
geo-fix.exe US        — включить (США)
geo-fix.exe DE        — включить (Германия)
geo-fix.exe --stop    — выключить
geo-fix.exe --cleanup — восстановить настройки после сбоя
geo-fix.exe --setup   — повторить мастер настройки
```

### Иконка в трее

При запуске в области уведомлений появляется иконка с кодом страны. Правый клик:
- **Переключить страну** → выбрать другую
- **Выключить** → остановить программу

## Как проверить

### 1. Включите VPN, затем запустите geo-fix

### 2. Проверьте на CreepJS
Откройте https://abrahamjuliot.github.io/creepjs/ и подождите 10–15 секунд.
- **Timezone**: `America/New_York` (для US), а не ваш реальный
- **Language**: `en-US`, а не `ru`
- Нигде не должно быть слова "Russia"

### 3. Проверьте WebRTC
Откройте https://browserleaks.com/webrtc
- Реальный IP не должен быть виден
- Только IP вашего VPN

### 4. Проверьте DNS
Откройте https://dnsleaktest.com → Extended test
- Не должно быть российских DNS-серверов

### 5. Откройте Google-сервисы
- https://notebooklm.google.com
- https://gemini.google.com
- Должны загрузиться без ошибок о регионе

## Про видеозвонки

**Google Meet, Zoom, Teams, Discord** — работают. WebRTC использует relay-режим (`iceTransportPolicy: 'relay'`): STUN-запросы не отправляются, реальный IP не утекает, а звонки идут через TURN-сервер. Задержка может быть чуть выше (обычно 10–50 мс), но качество остаётся нормальным.

## Про сертификат

При каждом запуске программа создаёт временный сертификат для работы HTTPS-прокси. Сертификат:
- Создаётся локально, никуда не отправляется
- Устанавливается в хранилище текущего пользователя Windows
- Закрытый ключ удаляется с диска через несколько секунд после старта
- Полностью удаляется при остановке программы

## Надёжность и восстановление

Программа автоматически восстанавливает все системные настройки при завершении. Даже если что-то пойдёт не так:

| Сценарий | Что произойдёт |
|----------|----------------|
| Нормальное выключение | Очистка мгновенная |
| Закрытие окна консоли | Очистка перед закрытием |
| Убили через Диспетчер задач | Watchdog-процесс обнаружит за 2 сек и восстановит |
| BSOD / отключение питания | При следующем входе в Windows запустится автоочистка |

Если интернет всё же перестал работать — запустите ярлык **Починить интернет** или `geo-fix.exe --cleanup`.

## Устранение неполадок

### Интернет не работает после сбоя
1. Запустите **Починить интернет** (ярлык) или `geo-fix.exe --cleanup`
2. Или вручную: Настройки Windows → Сеть → Прокси → Выключить "Использовать прокси-сервер"

### VPN не обнаружен
Программа предупредит при запуске. Можно продолжить, но реальный IP будет виден сайтам.

### Некоторые сайты не работают
Сайты с особо строгой политикой безопасности (банки, некоторые корпоративные порталы) могут не работать через прокси. Выключите geo-fix для таких сайтов.

### Антивирус ругается
Программа использует mitmproxy (open-source инструмент), который антивирусы иногда принимают за угрозу. Добавьте папку `geo-fix` в исключения.

## Удаление

1. Запустите `geo-fix.exe --cleanup`
2. Удалите папку `geo-fix`
3. Удалите ярлыки с рабочего стола

## Технические детали

- Локальный HTTPS-прокси (mitmproxy) на порту 8080, только localhost
- JS-инъекция с CSP nonce во все HTML-ответы
- Перехват Google Geolocation API на уровне прокси
- Автоматический перезапуск прокси при превышении 300 МБ RAM
- Поддержка Chrome, Edge, Firefox
- Исходный код: open-source
