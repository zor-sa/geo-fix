# Task 4: Firewall cleanup by prefix

## Description

This task adds test coverage for the prefix-based firewall rule cleanup introduced to fix R-7 (firewall cleanup misses rules if list changed between creation and removal).

The implementation already exists in `src/system_config.py`:
- `_list_firewall_rules_by_prefix(prefix)` — runs `netsh advfirewall firewall show rule name=all`, parses output, returns rule names starting with the given prefix
- `remove_firewall_rules()` — calls `_list_firewall_rules_by_prefix(FW_RULE_PREFIX)`, deletes each found rule, falls back to fixed-list if the query returns empty

The problem being solved: previously `remove_firewall_rules()` deleted only the fixed set of rule names it knew about (based on `BROWSER_EXES` × `STUN_PORTS`). If a rule was created with a different name (e.g., after a code change or manual addition), it would survive cleanup. The prefix-based approach finds all rules that start with `geo-fix-webrtc` regardless of how they were named.

The task is to write tests for this behavior so the implementation is protected from regressions.

## What to do

1. Write three unit tests in `tests/test_system_config.py` (create file if it does not exist, or add to existing test module):
   - `test_list_rules_parses_netsh_output` — verify that `_list_firewall_rules_by_prefix` correctly parses realistic `netsh` output and returns only the names that start with the prefix
   - `test_remove_by_prefix_deletes_found_rules` — verify that `remove_firewall_rules()` calls `netsh delete` for each rule found by the prefix query
   - `test_remove_by_prefix_fallback_on_parse_error` — verify that when `_list_firewall_rules_by_prefix` returns empty (simulated subprocess failure), `remove_firewall_rules()` falls back to deleting the fixed set of rule names

2. All tests must mock subprocess calls — no real `netsh` invocations. Tests must pass on non-Windows (Linux CI).

## TDD Anchor

Write these tests BEFORE touching any implementation. Run them, confirm they fail for the right reason, then implement or adjust until they pass.

- `tests/test_system_config.py::test_list_rules_parses_netsh_output` — mock `subprocess.run` to return realistic multi-rule netsh output; assert returned list contains exactly the names with matching prefix and excludes others
- `tests/test_system_config.py::test_remove_by_prefix_deletes_found_rules` — mock `_list_firewall_rules_by_prefix` to return two rule names; assert `subprocess.run` is called with `delete rule name=<each>` for both
- `tests/test_system_config.py::test_remove_by_prefix_fallback_on_parse_error` — mock `subprocess.run` so the `show rule name=all` call raises an exception; assert that `remove_firewall_rules()` still calls `subprocess.run` with delete for at least one known fixed-list rule name

**Current implementation state (`src/system_config.py`):**

`_list_firewall_rules_by_prefix(prefix)` (lines 536-553):
- Runs `netsh advfirewall firewall show rule name=all` with `timeout=30`
- Iterates lines, matches `line.strip().startswith("Rule Name:")`
- Extracts name after the first `:` and strips whitespace
- Returns names that `startswith(prefix)`
- On any exception: logs warning, returns `[]`

`remove_firewall_rules()` (lines 556-590):
- Guards with `sys.platform != "win32"` early return
- Calls `_list_firewall_rules_by_prefix(FW_RULE_PREFIX)` where `FW_RULE_PREFIX = "geo-fix-webrtc"`
- If `rules` is non-empty: calls `subprocess.run(["netsh", "advfirewall", "firewall", "delete", "rule", f"name={rule_name}"])` for each
- If `rules` is empty (query returned `[]`): iterates `BROWSER_EXES × STUN_PORTS` and tries to delete known fixed names

**Netsh output format to mock** (realistic example):
```
Rule Name:                            geo-fix-webrtc-chrome-udp-3478
----------------------------------------------------------------------
Enabled:                              Yes
Direction:                            Out
...
Rule Name:                            geo-fix-webrtc-msedge-udp-5349
----------------------------------------------------------------------
...
Rule Name:                            unrelated-other-rule
----------------------------------------------------------------------
```

**Edge cases:**
- Lines that start with `Rule Name:` but the name does not match the prefix must be excluded
- Empty output (no rules at all) → `_list_firewall_rules_by_prefix` returns `[]`
- `subprocess.run` raising `Exception` → `_list_firewall_rules_by_prefix` returns `[]` and `remove_firewall_rules` uses fallback

**Platform guard:** `remove_firewall_rules()` returns early on non-Windows (`sys.platform != "win32"`). To test the logic on Linux, either mock `sys.platform` or call `_list_firewall_rules_by_prefix` directly in some tests and test `remove_firewall_rules` with `unittest.mock.patch("sys.platform", "win32")`.

**Dependencies:** `unittest.mock` (stdlib), `pytest` — no new packages required.
