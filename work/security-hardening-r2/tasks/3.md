## Description

This task addresses two security/reliability issues found in analyst review: R-5 (VPN drop during operation goes unnoticed) and W-1 (watchdog can die without main process noticing).

**R-5 — VPN monitoring:** Currently, geo-fix only checks VPN status at startup. If the VPN disconnects mid-session, the user continues using a proxy that spoofs geo signals but exposes their real IP — which silently breaks the security model. A background monitor must detect this and alert the user immediately via stderr print and (when tray notification is added) tray toast.

**W-1 — Watchdog health:** The watchdog subprocess is spawned once at startup but is never checked again. If it dies (crash, OOM, etc.), the main process loses its safety net — if the main process then dies mid-session without triggering `_do_cleanup()`, system proxy settings and CA cert remain installed with no automatic recovery. The monitor must detect watchdog death and respawn it.

**Current state:** `_monitor_loop()` already exists in `main.py` and already includes both VPN checking and watchdog respawn logic — it was implemented partially in a previous session. The code is present but the tests are missing. This task's job is to verify the implementation is correct and complete, then add the required tests.

The monitor thread runs every 60 seconds using `stop_event.wait(timeout=60)`, so it exits cleanly when geo-fix shuts down.

## Details

**Files:**

- `src/main.py` — contains `_monitor_loop()` (lines 593–655). The function already implements VPN checking and watchdog respawn. Key variables:
  - `last_vpn` — tracks last known VPN state to suppress duplicate notifications
  - `_watchdog_proc` — global `subprocess.Popen` reference; checked via `.poll()` (returns `None` if alive)
  - `_spawn_watchdog(main_pid, state_file, session_tmpdir, session_id, stop_token)` — spawns watchdog subprocess, returns new `Popen`
  - The function uses `globals()["_watchdog_proc"] = ...` to update the global — this is the pattern already in place
  - VPN messages: `"⚠ VPN отключён! Реальный IP может быть виден."` and `"✓ VPN восстановлен."`

- `src/health_check.py` — contains `check_vpn_status() -> VpnStatus` and the `VpnStatus` enum. No changes needed.

- `src/tray.py` — `GeoFixTray` class. Currently has no `notify()` or `set_warning_icon()` methods. The current implementation only prints to stderr — tray toast is not yet wired up. Do not add tray notification methods in this task unless they are trivially testable; keep scope to monitoring logic and tests only.

**Dependencies:**

- No new packages required — `threading`, `subprocess`, `logging` are all stdlib
- `check_vpn_status` and `VpnStatus` are already imported in `main.py`

**Testing approach:**

- Tests should exercise the monitor logic by calling a stripped-down version of `_monitor_loop()` or by directly testing the branching logic with mocks
- Use `monkeypatch` or `unittest.mock.patch` to mock `check_vpn_status` return values and `_spawn_watchdog` calls
- Capture stderr output using `capsys` fixture to assert warning/restore messages
- To avoid running the full 60-second loop, extract the per-tick logic into a helper or test it by calling `_monitor_loop()` with a pre-set `stop_event` that stops after one tick

**Edge cases:**

- `VpnStatus.UNKNOWN` should not trigger notifications — treat as "no change"
- Watchdog respawn failure (exception from `_spawn_watchdog`) must be caught and logged; monitor loop must continue
- If `_watchdog_proc` is `None` (never spawned), skip the poll check entirely

**Implementation hints:**

- The existing `_monitor_loop()` already handles all the production logic. The main work is writing good tests that cover the state transitions
- Consider refactoring the per-tick logic out of the loop body into a `_monitor_tick()` helper function to make unit testing easier without running the full loop — but only if the refactor is cleaner than testing the loop directly
- For the duplicate-notification test: simulate two consecutive calls with `NOT_DETECTED` and assert `print` was called once total
