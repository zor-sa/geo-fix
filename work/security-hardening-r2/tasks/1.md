# Task 1: Delete CA key file after mitmproxy loads it

## Required Skills

Before starting, load: `/skill:code-writing` — [SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

After mitmproxy starts, the CA private key (`mitmproxy-ca.pem`) and PKCS12 bundle (`mitmproxy-ca.p12`) sit in the session tmpdir for the entire session. mitmproxy loads the key once at startup into `CertStore.default_privatekey` in memory and never re-reads from disk. Keeping the key on disk for hours is an unnecessary risk.

The fix is to delete the private key files immediately after mitmproxy confirms running, and delete the public cert after `install_ca_cert()` completes. Both functions (`delete_ca_key_files` and `delete_ca_public_cert`) and both call sites in `main.py` are already implemented. This task covers writing tests to verify correctness.

The implementation in `system_config.py` (lines 366–390) and `main.py` (lines 529–547) is already in place. The gap is test coverage — the four tests listed in the tech-spec do not exist yet.

## What to do

1. Write unit tests for `delete_ca_key_files(confdir)`:
   - Verify it deletes `mitmproxy-ca.pem`, `mitmproxy-ca-cert.cer`, `mitmproxy-ca.p12`
   - Verify it keeps `mitmproxy-ca-cert.pem` (public cert needed for install)
   - Verify it does not raise if files are already absent (idempotent)

2. Write unit test for `delete_ca_public_cert(confdir)`:
   - Verify it deletes `mitmproxy-ca-cert.pem`
   - Verify it does not raise if file is absent

3. Write integration test `test_mitmproxy_works_after_key_deletion`:
   - Start a real mitmproxy `Master` on a free port with a real confdir
   - Confirm proxy is listening
   - Call `delete_ca_key_files(confdir)` while master is running
   - Verify the proxy still accepts connections (private key lives in mitmproxy memory)
   - Shutdown master after test

4. Ensure all new tests pass on Linux (unit) and Windows (integration gate in CI).

## TDD Anchor

Tests to write before any further changes. Unit tests go in `test/unit/`, integration test in `test/`.

- `test/unit/test_delete_ca_key_files.py::test_delete_ca_key_files_removes_private_key` — creates tmpdir with all CA files, calls `delete_ca_key_files`, asserts `mitmproxy-ca.pem`, `mitmproxy-ca-cert.cer`, `mitmproxy-ca.p12` are gone
- `test/unit/test_delete_ca_key_files.py::test_delete_ca_key_files_keeps_public_cert` — same setup, asserts `mitmproxy-ca-cert.pem` still exists after call

- `test/unit/test_delete_ca_key_files.py::test_delete_ca_public_cert_removes_cert` — creates `mitmproxy-ca-cert.pem`, calls `delete_ca_public_cert`, asserts it is gone

- `test/test_integration_ca_key_deletion.py::test_mitmproxy_works_after_key_deletion` — integration: start real mitmproxy, delete key files while running, verify proxy still accepts connections
