## Description

Full-feature code quality audit across all source files created or modified in this feature: `system_config.py`, `proxy_addon.py`, `inject.js`, and `main.py`. Tasks 1–3 implement three defense layers (registry control, JS injection, proxy API intercept) that interact across components. This audit reviews the complete picture holistically — catching cross-component issues that per-task reviewers may miss.

Focus areas:
- Duplicate resource initialization (e.g., registry handles opened in multiple places)
- Shared resources compliance with Architecture decisions (threading.Lock, ProxyState, cleanup labels allowlist)
- Architectural consistency (retry/pending pattern applied uniformly, cleanup label validation, thread-safety of new code paths)
- Code quality: error handling coverage, logging discipline, dead code, naming consistency

## Details

**Files to audit:**

- `src/system_config.py` — new `disable_location_services()` / `restore_location_services()` functions, new cleanup label integration, `original_location_services` field in ProxyState. Check: read-before-write pattern, graceful fallback on missing registry path, label present in `_VALID_CLEANUP_LABELS` and dispatched in `_execute_cleanup_by_label()`, value validation against `{'Allow', 'Deny', None}`, non-Windows no-op guard

- `src/proxy_addon.py` — modified `response()` for non-target domain injection with CSP skip guard, new `request()` intercept for googleapis.com/geolocation. Check: exact host/path/method match (`host == 'www.googleapis.com'`, `path == '/geolocation/v1/geolocate'`, `method == 'POST'`), try/except wrapping intercept with log-and-passthrough on failure, CSP modification applied for non-target injection, randomized accuracy in range 40–80m, `script-src 'none'` skip guard

- `src/inject.js` — new `navigator.permissions.query` override scoped to geolocation only, geolocation-only payload for non-target injection. Check: permissions override only triggers when `name === 'geolocation'`, other permissions pass through to original, `.toString()` returns `[native code]`, geolocation-only payload doesn't include timezone/language/WebRTC overrides

- `src/main.py` — `disable_location_services()` call at startup, original value stored in ProxyState. Check: called before mitmproxy starts, not called on non-Windows, stored value flows into ProxyState correctly, error during disable logs warning and does not abort startup

**Cross-component checks:**
- ProxyState `original_location_services` field has backward-compatible default (`None`) so old state files still deserialize
- New cleanup label is in the validated allowlist (CWE-20 mitigation per architecture.md State Management section)
- winreg usage matches existing proxy registry pattern (HKCU, same error handling style)
- `disable_location_services()` called from main thread at startup — no lock contention with addon thread

**Architecture decision compliance to verify:**
- Decision 1: HKCU path used, graceful fallback if path absent
- Decision 3: exact host/path/method match (not substring)
- Decision 4: original value validation on restore, `None` → key deletion
- Decision 5: `script-src 'none'` skip guard implemented

**Report format:** JSON with fields `task` (4), `findings` (array of `{severity, file, issue, fix}`), `summary` (string)
