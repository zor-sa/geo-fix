## Details

**Acceptance criteria checklist — user-spec:**
- [ ] Windows Location Services disabled via HKCU registry at geo-fix startup (original value saved)
- [ ] Location Services restored to original state on stop (including crash recovery via cleanup resilience)
- [ ] If HKCU path fails — warning logged, startup continues without blocking
- [ ] JS injection of `navigator.geolocation` works on all domains (not just TARGET_DOMAINS)
- [ ] Non-target domains get geolocation-only injection (no timezone/language payload)
- [ ] `navigator.geolocation.getCurrentPosition()` returns fake coords on arbitrary domain
- [ ] `navigator.permissions.query({name: 'geolocation'})` returns `{state: 'granted'}` on all domains
- [ ] POST to `googleapis.com/geolocation/v1/geolocate` returns fake coords JSON from proxy
- [ ] Proxy intercept error → logged, request passes through unchanged (no page break)
- [ ] Location Services cleanup integrated into retry + `cleanup_pending.json` resilience
- [ ] Existing tests not broken

**Acceptance criteria checklist — tech-spec:**
- [ ] `disable_location_services()` reads and stores original value before writing `Deny`
- [ ] `restore_location_services()` writes exact original value back; `None` → key deleted; validates `{'Allow', 'Deny', None}`
- [ ] Non-Windows: Location Services functions are no-ops
- [ ] New cleanup label `"Location Services restore"` in `_VALID_CLEANUP_LABELS` and `_execute_cleanup_by_label()`
- [ ] `response()` injects geolocation-only JS on non-target domains (not full payload)
- [ ] `request()` intercepts exact `host == www.googleapis.com` + `path == /geolocation/v1/geolocate` + `POST` → fake JSON with randomized accuracy (40–80m)
- [ ] `navigator.permissions.query` override scoped to `name === 'geolocation'` only
- [ ] Non-target domains with `script-src 'none'` CSP → injection skipped, proxy intercept remains
- [ ] `request()` error handling: try/except, log on failure, pass through
- [ ] ProxyState `original_location_services` field with backward-compatible default
- [ ] All existing tests pass (no regressions)

Read each audit JSON. For any finding with severity `critical` or `high`: check if it has been addressed (look for commits, code changes, or decisions.md notes). If unresolved critical/high finding exists — QA does not pass; report what needs fixing.
