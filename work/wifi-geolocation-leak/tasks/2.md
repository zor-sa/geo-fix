## Description

This task closes the WiFi geolocation leak at the JS and proxy layers. Currently, `navigator.geolocation` is only overridden on Google TARGET_DOMAINS. Any other site can call the geolocation API and receive real device coordinates (obtained via WiFi AP scanning). Additionally, if Chrome's internal C++ geolocation stack fires a POST request to `googleapis.com/geolocation/v1/geolocate` before JS overrides execute, real MAC addresses are exfiltrated.

Two defenses are implemented here:

1. **Universal geolocation JS injection**: `response()` in `proxy_addon.py` is extended to inject a minimal geolocation-only payload on non-target domains (not the full timezone/language/WebRTC payload). A CSP skip guard prevents injection on pages with `script-src 'none'` or `require-trusted-types-for 'script'`, since those CSPs cannot be safely patched and the proxy-layer fallback covers them.

2. **Proxy-level geolocation API intercept**: `request()` in `proxy_addon.py` intercepts POST requests that exactly match `host == www.googleapis.com` + `path == /geolocation/v1/geolocate`. It returns a synthetic `200 OK` with fake coordinates and randomized accuracy (40–80m). This catches the API call even when JS injection fails or runs too late. Error handling wraps the intercept in try/except — on failure, log and pass through.

3. **`navigator.permissions.query` override**: Added to `inject.js` scoped only to `name === 'geolocation'`, returning `{state: 'granted'}`. Other permission queries are passed through to the real implementation unchanged.

The geolocation-only payload injected on non-target domains reuses the existing stealthDefine/disguiseFunction utilities from inject.js and the GF_LAT/GF_LON config vars. It does NOT include timezone, language, or WebRTC overrides — those are Google-specific.

3. Modify `response()` in `GeoFixAddon` to inject geolocation-only JS on non-target domains:
   - Remove the early return that currently skips non-target domains entirely
   - For non-target domains: check CSP for restrictive policy → if restrictive, skip injection; otherwise inject geolocation-only payload with nonce and modify CSP
   - For target domains: inject the full payload as before (no behavior change)
   - Geolocation-only injection uses the same nonce + CSP modification pattern as full injection

4. Modify `request()` in `GeoFixAddon` to intercept the geolocation API call:
   - Before the existing Accept-Language header rewrite, check for exact match: `flow.request.host == 'www.googleapis.com'` AND `flow.request.path == '/geolocation/v1/geolocate'` AND `flow.request.method == 'POST'`

   - Use `mitmproxy.http.Response.make(200, body_bytes, {"Content-Type": "application/json"})` to fabricate the response
   - Assign to `flow.response` to short-circuit the real request
   - Wrap in try/except Exception: on error, log with logger.error and allow request to pass through (do not re-raise)
   - The intercept must acquire `self._lock` to read preset coordinates

5. Add `navigator.permissions.query` override to `inject.js`:
   - In the Geolocation section (after the existing getCurrentPosition/watchPosition/clearWatch overrides)
   - Override `navigator.permissions.query` scoped to `name === 'geolocation'` only
   - For `name === 'geolocation'`: return `Promise.resolve({state: 'granted', onchange: null})`
   - For all other permission names: call original `navigator.permissions.query` with original arguments
   - Use disguiseFunction on the override
   - Guard with `if (navigator.permissions && navigator.permissions.query)`

6. Update `switch_preset()` in `GeoFixAddon` to also rebuild the geo-only payload when preset changes.

7. Write tests (see TDD Anchor).

## Details

**src/proxy_addon.py — what to change:**

- `_build_geo_only_payload(preset)` is a new module-level function (parallel to `_build_js_payload`). It builds a geolocation-only JS as an inline string containing only the geolocation + permissions overrides — no timezone, language, or WebRTC code. The inline JS should follow the same IIFE pattern as inject.js and substitute GF_LAT and GF_LON from the preset. The payload is intentionally small; focus on the override logic itself rather than replicating the full utility scaffolding from inject.js.

- `_has_restrictive_csp(csp_value: str) -> bool` checks for `script-src 'none'` as a directive value (not just substring — it must be the sole source in script-src) and `require-trusted-types-for 'script'` as a directive. Case-insensitive matching. A CSP like `script-src 'self' 'none'` is NOT restrictive — only `script-src 'none'` alone is.

- `response()` restructure: the current early return on non-target domain (line 160) becomes a branch. Non-target path: run the same HTML gating checks (content-type, size, status 200), then check CSP for restrictive policy before injecting geo-only payload. Target path: existing logic unchanged. The nonce generation and CSP modification logic is shared — extract to a helper or duplicate cleanly.

- `request()` intercept goes at the top of the method, before the Accept-Language assignment. Must use `flow.request.pretty_host` or `flow.request.host` — check which one mitmproxy's FakeRequest in tests exposes. Looking at the existing test fixture, `FakeRequest` has a `host` attribute directly, so match against `flow.request.host`. For the real response body: `{"location": {"lat": <preset.latitude>, "lng": <preset.longitude>}, "accuracy": <random 40-80>}`. Note the googleapis format uses `lng` (not `lon`).

- `GeoFixAddon.__init__` and `switch_preset()`: add `self._geo_only_payload = _build_geo_only_payload(preset)` alongside `self._js_payload`.

**src/inject.js — what to add:**

In the Geolocation section, after the existing clearWatch stealthDefine block (around line 196), add the permissions.query override. Guard it with `if (navigator.permissions && navigator.permissions.query)`. Store original in a local var before overriding. The override function checks `permissionDesc && permissionDesc.name === 'geolocation'` — if yes, return `Promise.resolve({state: 'granted', onchange: null})`; otherwise delegate to original with `origPermissionsQuery.call(navigator.permissions, permissionDesc)`.

**test/test_proxy_addon.py — test infrastructure:**

The existing `make_flow` fixture in `TestGeoFixAddon` uses `FakeRequest(host, path, headers)`. The existing `FakeRequest` constructor signature is `(host, path, headers)` but the class only sets `self.host`, `self.url`, and `self.headers` — it does NOT set `self.method` or `self.path` attributes. To test `request()` intercept you need a flow with `flow.request.method` and `flow.request.path` — extend `FakeRequest` to accept and store `method` and `path` as explicit attributes, or create a parallel `FakeRequestWithMethod` fixture for the new test class. Also ensure `flow.response = None` initially for all intercept tests (the existing `FakeFlow` may not set this).

**Integration tests:**

Integration tests start a real mitmproxy `Master` on a free port. Look at existing `test/test_integration_proxy.py` for the pattern — it uses `urllib.request` (not the `requests` library) to send HTTP through the proxy. Use the same `urllib.request` approach for consistency: configure a `ProxyHandler` with the proxy address and open connections via `urllib.request.urlopen`. **Edge cases:**

- Non-target domain with no CSP header at all → injection proceeds (no CSP = no restrictive policy)
- Non-target domain with CSP but no script-src → injection proceeds (will add nonce-based script-src)
- `navigator.permissions` doesn't exist (some older browsers) → guard prevents errors
- Intercept with preset switch in-flight → lock ensures consistent coordinates
- Existing test `test_response_skips_non_target_domain` (test_proxy_addon.py:172) asserts non-target responses are unchanged → must be updated to expect geo-only injection instead
- FakeFlow fixture always has FakeResponse set; tests for intercept skip cases (where flow.response should remain None) need a new fixture variant or explicit None assignment
