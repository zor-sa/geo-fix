# Task 1: Windows Location Services registry control

## Required Skills

## Description

Add `disable_location_services()` and `restore_location_services()` to `src/system_config.py` to block WiFi-based geolocation at the Windows registry level for the current user session.

Windows Location Services scans nearby WiFi access points and sends their MAC addresses to Google/Microsoft Location Service to determine real-world coordinates. This happens at the OS level, bypassing VPN and any JS-level overrides. Blocking it via HKCU registry (`{BFA794E4-F964-4FDB-90F6-51056BFE4B44}`) prevents MAC address exfiltration without requiring admin elevation.

This task implements only the registry operations and cleanup integration. Wiring into the startup sequence (calling these functions from main.py and updating ProxyState) is covered in Task 3.

## What to do

1. Implement `disable_location_services() -> Optional[str]`: reads the current registry value of `Value` under `HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\DeviceAccess\Global\{BFA794E4-F964-4FDB-90F6-51056BFE4B44}`, then writes `"Deny"` to it. Returns the original value (`"Allow"`, `"Deny"`, or `None` if the key did not exist). On non-Windows platforms: no-op, returns `None`. On registry errors: log warning, return `None` (graceful fallback ‚Äî does not raise).

2. Implement `restore_location_services(original: Optional[str]) -> None`: validates that `original` is in `{'Allow', 'Deny', None}`. If invalid, logs a warning and defaults to `"Deny"`. When `original` is `None`, deletes the registry value (key did not exist before). Otherwise writes back the original string. On non-Windows: no-op.

3. Write unit tests in `test/test_system_config_location.py` covering all cases from the TDD Anchor.

## Details

**Files:**

`src/system_config.py` ‚Äî the only file modified. Current state:
- Lines 200-220: `_get_registry_proxy_settings()` ‚Äî reference pattern for `winreg.OpenKey` + `winreg.QueryValueEx` with `FileNotFoundError` fallback.
- Lines 223-242: `set_wininet_proxy()` ‚Äî reference pattern for `sys.platform != "win32"` guard + `winreg.SetValueEx`.
- Lines 763-827: `cleanup()` ‚Äî Task 3 will wire the new functions here; this task does NOT touch `cleanup()`, `_execute_cleanup_by_label()`, `_VALID_CLEANUP_LABELS`, or `ProxyState`.

**Registry key details:**
- Hive: `HKEY_CURRENT_USER`
- Path: `SOFTWARE\Microsoft\Windows\CurrentVersion\DeviceAccess\Global\{BFA794E4-F964-4FDB-90F6-51056BFE4B44}`
- Value name: `Value`
- Value type: `REG_SZ`
- To disable: write `"Deny"`
- To restore "key never existed": call `winreg.DeleteValue(key, "Value")`

**Dependencies:** `winreg` ‚Äî stdlib on Windows, already imported conditionally elsewhere in the file (see `set_wininet_proxy`, `_find_browser_path`). Use the same pattern: import inside the function body after the `sys.platform` guard.

**Edge cases:**
- Registry key does not exist at all: `OpenKey` raises `FileNotFoundError` ‚Üí `disable_location_services()` should create it with `OpenKey(..., winreg.KEY_SET_VALUE)` or use `CreateKeyEx`; return `None` as original
- Registry value (`Value`) exists but key does not: impossible by definition
- `restore_location_services(None)` with key already deleted: `DeleteValue` raises `FileNotFoundError` ‚Üí catch and log, not a failure
- Value in registry could theoretically be something other than `"Allow"` or `"Deny"` (e.g., third-party software wrote an unexpected value). `disable_location_services()` should still return whatever it found, and `restore_location_services()` will then reject the invalid value and default to `"Deny"` ‚Äî correct behavior.

**Implementation hints:**
- Use `winreg.KEY_QUERY_VALUE | winreg.KEY_SET_VALUE` when opening the key for disable (need both read and write).
- Use `winreg.REG_SZ` as the type when calling `SetValueEx`.
- The `CreateKeyEx` / `OpenKey` with `KEY_SET_VALUE` will create intermediate keys if missing ‚Äî use `winreg.CreateKeyEx` if you need to guarantee the key exists before writing.
- For `restore_location_services(None)`, open with `KEY_SET_VALUE` and call `winreg.DeleteValue(key, "Value")` ‚Äî wrap in try/except `FileNotFoundError` since the value may already be gone.
- Follow the existing pattern exactly: `import winreg` inside the function after the `sys.platform` guard, use `with winreg.OpenKey(...) as key:` context manager.
