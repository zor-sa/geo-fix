# Task 7: Security Audit

## Required Skills

Before executing this task, load:
- `/skill:security-auditor` ‚Äî [skills/security-auditor/SKILL.md](~/.claude/skills/security-auditor/SKILL.md)

## Description

Full-feature security audit of the resource-optimization feature. The feature introduces three significant changes to the trusted proxy infrastructure: (1) replacing DumpMaster with a minimal Master setup, (2) adding a FlowCleanup addon that clears flow content after processing, and (3) adding a RAM watchdog with automatic proxy thread restart including CA certificate re-generation and re-installation.

Each of these changes touches security-sensitive code paths: the proxy restart sequence must maintain the CA key lifecycle guarantee (key files deleted from disk), DPAPI-encrypted state must remain consistent, CSP nonce injection must still function correctly with the minimal Master, and the restart mechanism must not open any new attack surface.

## What to do

1. Read all modified source files: `src/main.py`, `src/proxy_addon.py`, `src/presets.py`
2. Audit each security property listed in the Acceptance Criteria below
3. Check for any new attack surface introduced by the proxy restart mechanism
4. Check that no regressions were introduced in CSP nonce generation or injection logic
5. Write a structured audit report as `logs/working/task-7/security-auditor-1.json`

## Acceptance Criteria

- [ ] CA key lifecycle preserved on restart: new Master generates CA key ‚Üí key files deleted from disk after load ‚Üí public cert deleted after `certutil -addstore` ‚Üí old CA removed from cert store (`certutil -delstore`) before new one added
- [ ] DPAPI state encryption preserved: `ProxyState` is saved/loaded via `save_state`/`load_state` (DPAPI-backed) after each restart; `ca_thumbprint` updated in state before saving
- [ ] CSP nonce injection works with minimal Master: `GeoFixAddon.response()` generates a fresh `secrets.token_urlsafe(16)` nonce per response; `_modify_csp()` correctly appends nonce to `script-src`; no dependency on DumpMaster-specific addons removed
- [ ] No direct connection fallback during restart: system proxy remains set to `127.0.0.1:PORT` throughout restart; browser receives `ERR_PROXY_CONNECTION_REFUSED`, not a direct route; no requests leave the machine during the ~7 sec restart window
- [ ] FlowCleanup does not expose flow content: clearing `flow.request.content` and `flow.response.content` after processing reduces attack surface (GC reclaims sensitive HTTP bodies faster); no content is written to disk or logs
- [ ] No new attack surface from RAM monitor: memory reading uses read-only `GetProcessMemoryInfo` (no write, no exec); restart rate limit (max 3/hour, cooldown 10 min) prevents restart-loop amplification attacks
- [ ] Idle guard enforced before restart: `GeoFixAddon._last_flow_time` checked; restart deferred if flow activity within last 10 seconds; no in-flight requests torn down mid-processing
- [ ] GeoFixAddon instance reused across restart: same `addon` object (with locked `_preset` and `_js_payload`) passed to new Master; no credential material in addon state; thread-safe `_lock` preserved
- [ ] No sensitive data in new logging paths: RAM monitor logs only memory size and restart count; no flow content, headers, or user data logged
- [ ] Nonce entropy adequate: `secrets.token_urlsafe(16)` produces 128 bits of entropy ‚Äî sufficient for CSP nonce; generated per-response, not cached

## Details

**Security properties to verify in depth:**

**CA key lifecycle (Decision 4 in tech-spec):**
The restart sequence must: shutdown old master ‚Üí new `Master(opts)` auto-generates CA key+cert in `session_tmpdir` ‚Üí `certutil -delstore <old_thumbprint>` removes old CA ‚Üí `certutil -addstore` installs new CA cert ‚Üí `delete_ca_key_files(session_tmpdir)` deletes private key from disk ‚Üí `delete_ca_public_cert(session_tmpdir)` deletes public cert ‚Üí `state.ca_thumbprint = new_thumbprint` ‚Üí `save_state(state)`. Any deviation from this order creates a window where the key exists on disk or an orphaned CA remains in the cert store.

**DPAPI state encryption:**
`ProxyState` persists `ca_thumbprint` and session identifiers. After restart the thumbprint changes ‚Äî audit that the state is updated and saved before any further operations that depend on it. Confirm `save_state` uses DPAPI (check `src/system_config.py` if needed for reference).

**CSP nonce injection with minimal Master:**
With `DumpMaster` removed, `GeoFixAddon` is now the only response-processing addon (followed by `FlowCleanup`). Confirm no DumpMaster-bundled addon was inadvertently responsible for any security-relevant behavior (e.g., header stripping, response rewriting). Verify `_modify_csp()` logic for edge cases: missing `script-src`, `default-src` promotion, presence of `'unsafe-inline'` filtering.

**No direct-connection fallback (Decision 4 ‚Äî Safety verification):**
During restart, the system proxy setting (`127.0.0.1:PORT`) remains set. The port is not listening for ~7 seconds. Audit confirms this is the intended behavior: `ERR_PROXY_CONNECTION_REFUSED` does not cause browsers to bypass the proxy. Note: tech-spec marks this as requiring an integration test ‚Äî flag if that test is absent or incomplete.

**FlowCleanup attack surface:**
After `GeoFixAddon` processes a response, `FlowCleanup` nulls out request/response bodies. This is security-positive (reduces window for sensitive data in memory). Audit that `FlowCleanup` is strictly added *after* `GeoFixAddon` in the addon chain and cannot interfere with injection.

**RAM monitor ‚Äî no code execution paths:**
`GetProcessMemoryInfo` is a read-only Windows API call. The restart mechanism calls `master.shutdown()` and creates a new `Master` ‚Äî no shell commands, no `subprocess` calls, no `eval`. Audit for injection risk in any dynamic path (e.g., confdir path construction).

**Files to audit:**
- `src/main.py` ‚Äî `_start_mitmproxy()`, `_monitor_loop()` (RAM extension), restart sequence
- `src/proxy_addon.py` ‚Äî `GeoFixAddon`, `_modify_csp()`, `_generate_nonce()`, `FlowCleanup`
- `src/presets.py` ‚Äî `CountryPreset`, `is_target_domain()` (subdomain matching logic)

**Edge cases to check:**
- `is_target_domain()`: does `host == domain.lstrip(".")` prevent subdomain confusion? e.g., `evilgoogle.com` vs `.google.com`
- `_modify_csp()`: what if `script-src` already contains `'unsafe-inline'`? Nonce addition is still correct but flag for awareness
- Restart with no state file (crash between shutdown and save): watchdog recovery path
- Rate limit bypass: can a memory spike be induced faster than 10-min cooldown to force repeated restarts?
