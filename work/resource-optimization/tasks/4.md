# Task 4: Minor CPU optimizations in proxy_addon

## Required Skills

Before executing this task, load:

## Description

Two small CPU optimizations in the hot path of the proxy addon:

1. `_find_inject_position()` in `src/proxy_addon.py` currently calls `html_text.lower()` up front, creating a full copy of the HTML string in memory just to do case-insensitive tag searches. For large HTML pages this is wasteful — the copy is allocated, searched, and immediately discarded. The fix is to use `re.search()` with `re.IGNORECASE` (or `str.find()` after a targeted lower only on the slice being checked) so no full copy is needed.

2. `is_target_domain()` in `src/presets.py` currently loops over `TARGET_DOMAINS` with a Python `for` loop, calling `str.endswith()` once per domain. Python's built-in `str.endswith()` accepts a tuple of suffixes and tests all of them in a single C-level call — faster than a Python loop, especially as the domain list grows.

Both functions are in the hot path: `_find_inject_position()` is called on every HTML response from a target domain; `is_target_domain()` is called on every single request and response (including non-target domains). The changes are purely internal — no behaviour or interface changes.

This task depends on tasks 1 and 2 because by the time wave 2 runs, `proxy_addon.py` will already contain the new `FlowCleanup` class (task 2). Edits here must not conflict with those additions.

## What to do

1. In `src/proxy_addon.py`, rewrite `_find_inject_position()` to eliminate the `lower = html_text.lower()` full-string copy. Use `re.search()` with `re.IGNORECASE` to find `<head`, `<html`, and `<!doctype` positions without creating a lowercase copy of the entire HTML string.

2. In `src/presets.py`, pre-compute a module-level tuple from `TARGET_DOMAINS` for use in `is_target_domain()`. Replace the `for domain in TARGET_DOMAINS` loop with a single `host.endswith(tuple_of_suffixes)` call. Keep the existing exact-match check for bare domain names (e.g. `google.com` matching `.google.com`).

3. Add `import re` to `src/proxy_addon.py` if not already present.

4. Verify all existing tests still pass — no behaviour changes are permitted.

**Edge cases:**
- HTML with no `<head>`, no `<html>`, no `<!doctype>` — must return 0 (unchanged)
- `<head` with attributes like `<head lang="en">` — inject position must be after the closing `>` of the opening tag
- `is_target_domain("")` — empty string must return False without raising
- Bare domain match: `google.com` (without leading dot) matching `.google.com` entry — the existing `host == domain.lstrip(".")` branch must be preserved

**Implementation hints for `_find_inject_position()`:**
- `re.search(r"<head[\s>]", html_text, re.IGNORECASE)` gives a match object; use `.start()` to find where the tag begins, then `html_text.find(">", match.start())` to find the closing `>` — same logic as before but without the full `.lower()` copy.
- Order of searches must remain: `<head` first, then `<html`, then `<!doctype`.
- `re.search` returns `None` if no match — check before using `.start()`.

**Implementation hints for `is_target_domain()`:**
- Add at module level: `_TARGET_DOMAINS_TUPLE = tuple(TARGET_DOMAINS)` — computed once at import time.
- In `is_target_domain()`, replace the loop with: `return host.endswith(_TARGET_DOMAINS_TUPLE) or host in {d.lstrip(".") for d in TARGET_DOMAINS}` — or pre-compute the bare-domain set as well.
- The existing `host = host.lower()` line at the top of the function must stay.
