# Task 9: Pre-deploy QA

## Required Skills

Before executing this task, load:
- `/skill:pre-deploy-qa` — [SKILL.md](~/.claude/skills/pre-deploy-qa/SKILL.md)

## Description

Pre-deploy acceptance testing for the resource-optimization feature. Run the full test suite (unit + integration + E2E), verify all acceptance criteria from user-spec and tech-spec, confirm no regressions.

This is the final wave after audits (tasks 6, 7, 8) — everything is already implemented and quality-reviewed. The QA task ensures the implementation meets user requirements and technical criteria.

## What to do

1. Run the full test suite: `pytest test/ -x -v`
2. Go through each acceptance criteria item from user-spec and tech-spec — mark as verified or failed
3. Record results in the QA report
4. If any criterion is not met — record as a blocker

## Acceptance Criteria

**From user-spec (resources):**
- [ ] Baseline RAM at startup does not exceed 150MB (main process + watchdog)
- [ ] RAM stable after 8 hours — Private Working Set does not grow more than 20% from initial
- [ ] Average CPU under active browsing (15+ Google tabs) does not exceed 5% over a 5-minute window

**From user-spec (optimization):**
- [ ] Completed flows removed from proxy memory after processing — stored flow count does not grow over time
- [ ] WebSocket message history trimmed after processing — no more than 1 last message stored
- [ ] On RAM threshold breach (300MB), proxy auto-restarts only if no active traffic (>10 sec idle); cooldown between restarts at least 10 minutes; max 3 restarts per hour

**From user-spec (functionality preservation):**
- [ ] All existing tests (unit, integration, E2E) pass without changes
- [ ] Accept-Language header rewriting works on target domains and any other domains
- [ ] JS injection works on target domains — injected script present in Google HTML responses
- [ ] CSP header contains nonce of injected script — browser does not block execution
- [ ] On main process crash, watchdog performs cleanup within 10 seconds
- [ ] Country switching via tray menu switches preset without proxy restart

**From tech-spec (technical criteria):**
- [ ] mitmproxy starts with base Master class, not DumpMaster — unused addons not loaded
- [ ] Only essential addons loaded: Core, Proxyserver, NextLayer, TlsConfig, KeepServing, ErrorCheck, GeoFixAddon, FlowCleanup
- [ ] FlowCleanup added after GeoFixAddon — verified by unit test checking addon order
- [ ] FlowCleanup clears flow content after processing — flow.request.content and flow.response.content set to None/empty
- [ ] WebSocket message history trimmed to <=1 message per connection
- [ ] RAM monitoring triggers at 300MB Private Working Set with 10-min cooldown, max 3/hour
- [ ] Proxy restart only occurs after 10+ seconds of no flow activity (idle guard)
- [ ] Proxy restart reuses the same GeoFixAddon instance (preserves current preset and JS payload cache)
- [ ] CA certificate re-generated, re-installed, and key deleted on proxy restart — HTTPS works after restart
- [ ] During proxy restart, no HTTP requests leave the machine (browser gets connection refused, not direct fallback)
- [ ] No regressions in existing test suite (unit + integration + E2E)
- [ ] Dumper addon not loaded (no stdout output per flow)
