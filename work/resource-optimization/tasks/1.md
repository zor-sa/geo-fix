# Task 1: Replace DumpMaster with minimal Master

## Required Skills

Before starting, load:
- `/skill:code-writing` — [SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

`_start_mitmproxy()` in `src/main.py` currently initializes mitmproxy via `DumpMaster`, which loads ~35 default addons including Dumper (stdout per-flow printer), SaveHar, ClientPlayback, ScriptLoader, and ~26 others that geo-fix never uses. Each addon registers hooks that are checked on every flow event — CPU overhead per request scales linearly with addon count.

This task replaces `DumpMaster` with the base `mitmproxy.master.Master` class and manually adds only the 6 addons required for HTTPS proxying: `Core`, `Proxyserver`, `NextLayer`, `TlsConfig`, `KeepServing`, `ErrorCheck`. `GeoFixAddon` continues to be added after these. The result is ~80% fewer hooks fired per flow and no stdout logging per flow from the removed `Dumper` addon.

See Decision 1 in `tech-spec.md` for full rationale and alternatives considered.

## What to do

1. **Replace the import** in `_start_mitmproxy()` in `src/main.py`:
   - Remove: `from mitmproxy.tools.dump import DumpMaster`
   - Add: `from mitmproxy.master import Master`
   - Add imports for the 6 essential addons: `Core` from `mitmproxy.addons.core`, `Proxyserver` from `mitmproxy.addons.proxyserver`, `NextLayer` from `mitmproxy.addons.next_layer`, `TlsConfig` from `mitmproxy.addons.tlsconfig`, `KeepServing` from `mitmproxy.addons.keepserving`, `ErrorCheck` from `mitmproxy.addons.errorcheck`

2. **Replace `DumpMaster` instantiation** inside `run_proxy()`:
   - Replace `master = DumpMaster(opts)` with `master = Master(opts)`
   - Replace `master.addons.add(addon)` with a single `master.addons.add()` call that adds all addons in order: `Core()`, `Proxyserver()`, `NextLayer()`, `TlsConfig()`, `KeepServing()`, `ErrorCheck()`, then the existing `addon` (GeoFixAddon instance)

3. **Verify no other references** to `DumpMaster` remain in `src/main.py`

4. **Return `(thread, master)` from `_start_mitmproxy()`** instead of just `thread`:
   - Task 3 needs the `master` reference to perform a controlled restart of the proxy
   - Update the return statement and all call sites in `src/main.py` that unpack the return value

    ...

**Target shape after change:**
```python
from mitmproxy.options import Options
from mitmproxy.master import Master
from mitmproxy.addons.core import Core
from mitmproxy.addons.proxyserver import Proxyserver
from mitmproxy.addons.next_layer import NextLayer
from mitmproxy.addons.tlsconfig import TlsConfig
from mitmproxy.addons.keepserving import KeepServing
from mitmproxy.addons.errorcheck import ErrorCheck

def run_proxy():
    kwargs = dict(listen_host=PROXY_HOST, listen_port=port)
    if confdir:
        kwargs["confdir"] = confdir
    opts = Options(**kwargs)
    master = Master(opts)
    master.addons.add(Core(), Proxyserver(), NextLayer(), TlsConfig(),
                      KeepServing(), ErrorCheck(), addon)
    ...
    master.run()
```

**`GeoFixAddon` interface (from `src/proxy_addon.py`):** Implements `request(flow)` and `response(flow)` hooks. No changes needed — it slots in as a standard mitmproxy addon regardless of whether the host is `DumpMaster` or `Master`.

**`src/presets.py`:** Read-only context. Contains `CountryPreset` dataclass and `PRESETS` dict. No changes needed.

**PyInstaller note (from tech-spec Risks):** Switching from `mitmproxy.tools.dump` to explicit addon imports changes the import graph. If a CI build step exists, new `--hidden-import` flags may be needed in `build/build.py`. Note this in the decisions.md report if encountered.

**Edge cases:**
- `KeepServing` prevents the event loop from exiting prematurely when there are no active connections — required for long-running proxy
- `ErrorCheck` reports startup errors (e.g., port already in use) before the proxy starts accepting connections — needed for the 15-second startup wait loop in `_start_mitmproxy()`
- `NextLayer` handles protocol detection (HTTP/1.1, HTTP/2, WebSocket, CONNECT tunneling) — required for correct HTTPS interception
