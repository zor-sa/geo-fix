# Task 10: Fix setup wizard — skip warning, console parity, per-session CA adaptation

## Required Skills
Before starting, load: /skill:code-writing

## Description
The setup wizard has three defects. First, the "Пропустить настройку" button silently marks setup as complete without installing any CA certificate, breaking HTTPS interception without any user warning — users must explicitly confirm they understand the consequences. Second, the console-mode wizard (`_run_console_wizard`) skips the firewall step and DNS instructions that the GUI wizard provides, creating a feature gap for headless installs. Third, `setup_wizard.py` imports and calls `install_ca_cert()` directly — since task 1 moved CA installation to per-session in `main.py`, the wizard should no longer call it (removing a redundant install that would use the wrong confdir).

## What to do

1. Fix the "Пропустить" button in `_run_gui_wizard()`:
   - Replace the current lambda `command=lambda: (mark_setup_complete(), root.destroy())` with a proper handler function.
   - The handler must call `messagebox.askokcancel()` (or `messagebox.askyesno()`) with a warning dialog:
     - Title: `"Пропустить настройку?"`
     - Message: `"Без настройки HTTPS-перехват работать не будет.\n\nBраузер будет получать ошибки сертификата для всех сайтов.\n\nВы уверены, что хотите пропустить?"`
   - Only if the user confirms: call `mark_setup_complete()` and `root.destroy()`.
   - If the user cancels: do nothing (return without destroying root).

2. Remove the direct CA install call from the wizard:
   - Remove `install_ca_cert` from the import in `setup_wizard.py`.
   - In `step_cert()` in `_run_gui_wizard()`, replace the `install_ca_cert()` call with a status message explaining that the CA cert will be installed automatically when geo-fix starts.
   - Update the displayed text to inform the user that each session generates a fresh CA (per-session security).

3. Add firewall step to `_run_console_wizard()`:
   - After the current step 1 (cert/info), add a firewall prompt matching the GUI wizard's step 2:
     ```python
     answer = input("\nШаг 2: Установить правила файрвола для WebRTC-защиты? (требует права администратора) [y/N]: ")
     if answer.strip().lower() == "y":
         if create_firewall_rules():
             print("  ✓ Правила файрвола установлены")
         else:
             print("  ⚠ Не удалось установить правила файрвола. Базовая защита активна.")
     else:
         print("  Файрвол пропущен. Базовая защита WebRTC активна.")
     ```

4. Add DNS instructions to `_run_console_wizard()`:
   - After the firewall step, add step 3 matching the GUI wizard's DNS step:
     ```
     Шаг 3: Настройте «Безопасный DNS» в браузере:
       Chrome: chrome://settings/security → Безопасный DNS → Включить
       Firefox: about:preferences#privacy → DNS через HTTPS → Включить
     ```

5. Update imports in `setup_wizard.py`: remove `install_ca_cert` from the import line; keep `create_firewall_rules`.

## Details
Current skip button (line 181 in `setup_wizard.py`):
```python
skip_btn = tk.Button(btn_frame, text="Пропустить настройку",
                     command=lambda: (mark_setup_complete(), root.destroy()))
```
This lambda fires both `mark_setup_complete()` and `root.destroy()` immediately without any confirmation. Replace with a named function that calls `messagebox.askokcancel()` first.

Current `_run_console_wizard` only does two things: attempts CA cert install and prints generic DNS instructions. It skips firewall entirely and the DNS instructions it prints do not match the detailed GUI text. After this task, console wizard must cover all three wizard steps.

The `install_ca_cert` import in line 19 (`from .system_config import (MITMPROXY_CA_CERT, create_firewall_rules, install_ca_cert,)`) must be cleaned up. `MITMPROXY_CA_CERT` may also be removable if the wizard no longer references it — check all usages.

Note: `_run_gui_wizard()` currently returns `success.get("firewall", False)` on line 194 — this return value is used by `run_setup_wizard`. Verify the return value semantics remain sensible after removing direct CA install.
