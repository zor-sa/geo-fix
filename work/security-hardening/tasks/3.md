# Task 3: Safe Firefox backup with copy semantics and enterprise_roots cleanup

## Required Skills
Before starting, load: /skill:code-writing

## Description
The current Firefox `user.js` backup uses `Path.rename()` which is an atomic move — if the process dies after rename but before writing the new file, the original user.js is gone permanently. This task replaces both backup and restore with copy-based semantics: `shutil.copy2()` for backup (original stays in place), and copy+unlink for restore (so a crash between the two leaves the backup intact). Additionally, `security.enterprise_roots.enabled=true` is currently written into `user.js` but never explicitly reverted — even if the user had that pref set to `false`. This task ensures the pref is restored to its pre-proxy value on every cleanup.

## What to do

1. **Modify `set_firefox_proxy()` in `src/system_config.py`** (lines 195–229):
   - Replace `prefs_file.rename(backup_path)` (line 208) with `shutil.copy2(str(prefs_file), backup_path)`
   - After the copy, do NOT delete the original — it stays in place
   - The new user.js content (proxy prefs) is then written on top of the original using `prefs_file.write_text(...)`, replacing it
   - Add `import shutil` at the top if not already present
   - The backup now serves as a safety copy, not the only copy of the original

2. **Modify `unset_firefox_proxy()` in `src/system_config.py`** (lines 232–249):
   - Replace `Path(backup_path).rename(prefs_file)` (line 242) with:
     ```python
     shutil.copy2(backup_path, str(prefs_file))
     Path(backup_path).unlink()
     ```
   - This ensures: if the process crashes after copy but before unlink, the backup file still exists and can be used again

3. **Handle `enterprise_roots` revert in `unset_firefox_proxy()`**:
   - After restoring the original user.js (from backup or by unlinking geo-fix's user.js), explicitly check if `security.enterprise_roots.enabled` is present in the restored file
   - If the original user.js did NOT have `security.enterprise_roots.enabled=true`, verify it is absent from the restored file
   - Since the original is restored verbatim via `shutil.copy2`, the pref automatically reverts — but add a verification log line: `logger.info("enterprise_roots pref reverted with user.js restore")`
   - If there was no original (no backup path), and the created user.js is being unlinked, `enterprise_roots` is removed as part of the whole file deletion — no extra step needed

4. **Refactor proxy content writing** to keep the logic clean:
   - If a backup exists: write only the geo-fix proxy prefs to `user.js` (the original content is preserved in the backup, not prepended). This changes from the current behavior of prepending original content — the current approach is fine but make sure the intent is clear in comments.
   - Actually: keep current behavior of prepending original content (so Firefox still has user's original prefs during geo-fix session). Only change is the backup mechanism (copy vs rename).

5. **Add `import shutil`** to `src/system_config.py` imports section (line 7–16).

## Details

**Current `set_firefox_proxy()` backup code** (lines 206–208):
```python
backup_path = str(prefs_file.with_suffix(".js.geo-fix-backup"))
prefs_file.rename(backup_path)
```
`Path.rename()` is a filesystem rename (atomic move). The original file is moved to `backup_path`. The original path is now empty. If the process dies before `prefs_file.write_text(...)` executes, the user's user.js is gone — only recoverable from `backup_path` if it exists and the user knows to look for it.

**Current `unset_firefox_proxy()` restore code** (line 242):
```python
Path(backup_path).rename(prefs_file)
```
Same problem in reverse: if process dies after rename starts but before it completes (rare but possible on network drives), or if rename is interrupted, user.js could be lost.

**`enterprise_roots` handling**: The pref `security.enterprise_roots.enabled=true` is written into the geo-fix user.js on every `set_firefox_proxy()` call (line 218). On restore, `unset_firefox_proxy()` either restores the original backup (which lacks this pref if the user never had it) or deletes the geo-fix user.js. Both paths correctly remove the pref — but only if the restore actually runs. Task 2 ensures all exit paths call cleanup. Task 3 ensures the restore mechanism itself is safe.

**No changes needed to `PROXY_HOST`/`PROXY_PORT` in user.js content** — these are still correct constants. The proxy port will be made dynamic in Task 5, at which point `set_firefox_proxy()` will need to accept a `port` parameter. Do NOT pre-empt Task 5 here.

**`shutil` import**: currently `import shutil` is not present in `system_config.py` (imports: json, logging, os, subprocess, sys, tempfile, dataclasses, pathlib, typing). Must be added.
