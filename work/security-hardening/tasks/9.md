# Task 9: Add SHA-256 verification and read-only launchers to install.bat

## Required Skills
Before starting, load: /skill:code-writing

## Description
`install.bat` downloads two external files — the Python embedded zip and `get-pip.py` — without verifying their integrity. A MITM attacker or compromised DNS server could serve a trojaned payload and it would be silently executed. Both downloads must be verified against pinned SHA-256 hashes using PowerShell's `Get-FileHash`. Additionally, `get-pip.py` must be pinned to a versioned URL (`https://bootstrap.pypa.io/pip/{version}/get-pip.py`) rather than the rolling `https://bootstrap.pypa.io/get-pip.py`. Finally, the generated `.bat` launcher file must be set read-only with `attrib +R` to prevent malware persistence via launcher replacement.

## What to do

1. Determine the pinned SHA-256 hashes:
   - Python 3.12.7 embedded AMD64 zip: compute or look up the official hash from `https://www.python.org/downloads/release/python-3127/`. Hash should be embedded as a comment with the verification URL.
   - pip 24.x (latest stable at time of implementation): pin to versioned URL `https://bootstrap.pypa.io/pip/24.3.1/get-pip.py` (or current latest stable) and compute its SHA-256.

2. Add hash verification for Python zip in `install.bat`:
   ```batch
   :: Pinned SHA-256 (verify at https://www.python.org/downloads/release/python-3127/)
   set "PYTHON_ZIP_HASH=<sha256-hex>"
   ```
   After the `Invoke-WebRequest` download, add a verification step:
   ```batch
   powershell -Command "& { $h = (Get-FileHash '%INSTALL_DIR%\python-embed.zip' -Algorithm SHA256).Hash; if ($h -ne '%PYTHON_ZIP_HASH%') { Write-Error 'Hash mismatch: python-embed.zip'; exit 1 } }"
   if errorlevel 1 (
       echo ОШИБКА: Хэш Python zip не совпадает. Возможно, файл повреждён или подменён.
       del "%INSTALL_DIR%\python-embed.zip" 2>nul
       pause
       exit /b 2
   )
   ```

3. Pin `get-pip.py` to versioned URL and add hash verification:
   - Change `PIP_URL` from `https://bootstrap.pypa.io/get-pip.py` to `https://bootstrap.pypa.io/pip/24.3.1/get-pip.py` (or current stable version).
   - Add `PIP_HASH` variable with pinned SHA-256.
   - After download, verify hash; on mismatch: print error, delete file, exit /b 2.

4. Set launcher read-only after creation:
   - After the `echo` lines that write `geo-fix.bat`, add: `attrib +R "%INSTALL_DIR%\geo-fix.bat"`

5. Add comments above each hash variable pointing to the verification URL so maintainers know where to find updated hashes when bumping versions.

## Details
Current `install.bat` structure: step [2/6] downloads Python zip, step [3/6] downloads `get-pip.py`. The hash verification blocks must be inserted immediately after each `Invoke-WebRequest` command, before `Expand-Archive` or `python.exe` invocation respectively. Exit code `/b 2` distinguishes hash failure from other errors (`/b 1`).

The `Get-FileHash` PowerShell cmdlet returns the hash in uppercase hex. The comparison in the PowerShell `-ne` check should be case-insensitive or store the hash in uppercase. Use: `if ($h.ToUpper() -ne '%PYTHON_ZIP_HASH%'.ToUpper())`.

For the `attrib +R` step, place it immediately after the final `echo` line that writes `geo-fix.bat` (currently line 70). The launcher is not currently set read-only — this is a one-line addition.

Pinned hashes must be real values. Look up the official Python 3.12.7 hash from the Python downloads page. For get-pip.py, download the versioned URL and compute `certutil -hashfile get-pip.py SHA256` or PowerShell `Get-FileHash`.
