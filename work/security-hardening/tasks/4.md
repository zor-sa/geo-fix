# Task 4: Implement watchdog subprocess with session-aware cleanup

## Required Skills
Before starting, load: /skill:code-writing

## Description
When geo-fix is killed with `taskkill /F` or during a BSOD/power-loss, no Python cleanup runs тАФ the system is left with WinINET proxy enabled, the CA cert installed, and Firefox modified. This task creates a standalone `watchdog.py` process that is spawned before any system modifications and monitors the main process PID. If the main process dies unexpectedly, the watchdog calls `system_config.cleanup()` using the same state file that was saved at startup. A boot-time ONLOGON scheduled task provides a second layer of fallback for power-loss scenarios. The watchdog uses a session ID to verify it is cleaning up the correct session, preventing races when geo-fix is restarted immediately after a crash.

## What to do

1. **Create `src/watchdog.py`** as a standalone Python script:
   - Accept CLI args: `main_pid`, `state_file_path`, `session_tmpdir_path`, `session_id`, `stop_token`
   - Loop every 2 seconds:
     a. Check if stop flag file (`.geo-fix-watchdog-stop` in tmpdir) exists and contains matching `stop_token` тЖТ exit cleanly
     b. Check if `main_pid` is alive: on Windows use `ctypes.windll.kernel32.OpenProcess(SYNCHRONIZE, False, pid)` and close the handle; on other platforms use `os.kill(pid, 0)`
     c. If main process is dead:
        - Load state from `state_file_path` using `system_config.load_state()`
        - Verify `state.session_id == session_id` тАФ if mismatch, log and exit without cleanup
        - Call `system_config.cleanup(state)`
        - Call `shutil.rmtree(session_tmpdir_path, ignore_errors=True)` (belt-and-suspenders, cleanup() also does it)
        - Remove ONLOGON scheduled task via `schtasks /delete /tn "geo-fix-cleanup" /f`
        - Exit
   - Register ONLOGON scheduled task at startup: `schtasks /create /sc ONLOGON /tn "geo-fix-cleanup" /tr "<exe> --cleanup" /rl LIMITED /f`
     - `<exe>` is `sys.argv[0]` (the geo-fix executable path, passed as CLI arg or inferred)
   - Handle `KeyboardInterrupt` and unexpected exceptions gracefully (log, continue loop or exit depending on severity)

2. **Integrate watchdog into `main.py`**:
   - After step 7 in the startup sequence (CA cert installed, thumbprint recorded, state NOT yet saved):
     - Generate `stop_token = secrets.token_hex(32)`
     - Save state file first (so watchdog can load it): call `save_state(state)` with partial state
     - Spawn watchdog: `subprocess.Popen([sys.executable, watchdog_path, str(os.getpid()), str(STATE_FILE), session_tmpdir, session_id, stop_token], creationflags=CREATE_NEW_PROCESS_GROUP | CREATE_NO_WINDOW)`
     - Store the watchdog `Popen` object in a module-level variable `_watchdog_proc`
   - Add a helper `_get_watchdog_path() -> str` that:
     - In normal execution: returns `Path(__file__).parent / "watchdog.py"` (relative to `main.py`)
     - In PyInstaller bundle (`sys.frozen` is True): extracts `watchdog.py` from `sys._MEIPASS / "src" / "watchdog.py"` to a temp file and returns that path

3. **Signal watchdog to stop on clean shutdown** in `_do_cleanup()` in `main.py`:
   - Write `stop_token` to the stop flag file: `Path(session_tmpdir) / ".geo-fix-watchdog-stop"`
   - This must happen BEFORE `cleanup()` deletes the session tmpdir
   - Add `import secrets` at the top of `main.py`

4. **Remove ONLOGON scheduled task on clean shutdown** in the shutdown sequence (after watchdog is signaled):
   - Run `schtasks /delete /tn "geo-fix-cleanup" /f` (idempotent, ignore failure)

5. **Update `build/build.py`** to bundle `watchdog.py`:
   - Add `--add-data "src/watchdog.py;src"` to the PyInstaller command

6. **Handle `sys.path` in `watchdog.py`**:
   - At the top of `watchdog.py`, add `sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))` so it can import `src.system_config`
   - In PyInstaller context, `sys.executable` is the .exe itself and `sys._MEIPASS` provides the extraction root

## Details

**Current startup sequence in `main()`** (lines 239тАУ260): proxy is set and state saved BEFORE mitmproxy starts. The watchdog must be started AFTER state is saved (so it can load state on crash) but BEFORE system changes (WinINET proxy, Firefox) so it can clean them up if main dies mid-setup. Exact ordering per tech-spec: watchdog starts at step 8, after CA install (step 7) and before WinINET proxy (step 10).

**`CREATE_NEW_PROCESS_GROUP` flag value**: `0x00000200` on Windows. Also use `CREATE_NO_WINDOW = 0x08000000`. These must be imported from `subprocess` or defined as constants since `subprocess.CREATE_NEW_PROCESS_GROUP` is available in Python 3.7+ on Windows.

**PID alive check on Windows**: `OpenProcess` with `SYNCHRONIZE` (0x00100000) тАФ if it returns a non-null handle, process is alive; close the handle immediately. This works without elevation for processes owned by the same user. `WaitForSingleObject(handle, 0)` returns `WAIT_OBJECT_0` (0) if the process has exited, `WAIT_TIMEOUT` (258) if still alive.

**PyInstaller compatibility note**: `sys.executable` in a PyInstaller bundle IS the .exe. To run `watchdog.py` as a script, use `[sys.executable, watchdog_path, ...]` тАФ the exe acts as a Python interpreter for script arguments because PyInstaller embeds a Python interpreter. The `watchdog_path` must be extracted from `sys._MEIPASS` to a writable temp location since MEIPASS is read-only in some configurations.

**Stop token security**: The stop token prevents a malicious process (running as the same user) from faking a clean shutdown. The token is a random 32-byte hex string generated with `secrets.token_hex(32)`. It is passed to the watchdog as a CLI argument and written to the stop flag file only on clean shutdown. Since the watchdog checks for a file in the session tmpdir (which has restricted ACL from Task 1), this provides reasonable protection.

**ONLOGON task stale state detection**: When `--cleanup` runs at login, it should treat state files older than 24 hours as stale (per tech-spec watchdog design). Add this check in `_handle_cleanup()` or in `cleanup()`: if `state.timestamp` is more than 24 hours ago, proceed with cleanup unconditionally.
