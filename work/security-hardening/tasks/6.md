# Task 6: Encrypt state file with DPAPI user-scope protection

## Required Skills
Before starting, load: /skill:code-writing

## Description
The current state file is plain JSON at `.geo-fix-state.json`, readable by any process running as the same user and trivially modifiable to inject a malicious proxy server address. This task replaces plaintext persistence with a DPAPI-encrypted blob using `CryptProtectData` / `CryptUnprotectData` via ctypes, bound to the current Windows user with fixed application entropy `b"geo-fix-state-v1"`. The encrypted blob is written atomically. On load, any failure to decrypt (tampered bytes, wrong user, corruption) must fall back to clean defaults with a logged warning and best-effort cleanup â€” never silently apply attacker-controlled values.

## What to do

1. Add `_dpapi_encrypt(plaintext: bytes) -> bytes` and `_dpapi_decrypt(ciphertext: bytes) -> bytes` in `system_config.py` using ctypes `crypt32.CryptProtectData` / `CryptUnprotectData` with `DATA_BLOB` structures and `LocalFree` on output. Use `optionalEntropy=b"geo-fix-state-v1"`. On non-Windows, pass through plaintext unchanged with a warning log.
2. Extend `ProxyState` with three new optional fields: `session_id: Optional[str] = None`, `ca_thumbprint: Optional[str] = None`, `session_tmpdir: Optional[str] = None`. Update `from_json` schema validation to accept them.
3. Modify `save_state(state)` to serialize to JSON bytes, encrypt via `_dpapi_encrypt`, and write the raw ciphertext (not text) atomically via temp-file-rename. File extension can stay `.geo-fix-state.json` or change to `.geo-fix-state.bin` â€” document the choice in decisions.md.
4. Modify `load_state()` to read raw bytes, call `_dpapi_decrypt`, then JSON-parse. On any exception during decrypt or parse: log `warning("State file rejected: %s", e)`, call `delete_state()`, and return `None` (clean defaults path in caller).
5. Ensure `delete_state()` removes the file regardless of extension change.
6. Update `cleanup()` to also call `uninstall_ca_cert(state.ca_thumbprint)` and delete `state.session_tmpdir` (tasks 1-2 provide those functions; call them here too so cleanup is self-contained).

## Details
Current `save_state` uses `write_text` (UTF-8 JSON). `load_state` uses `read_text`. Both must switch to byte-level `read_bytes` / `write_bytes`. The `DATA_BLOB` ctypes structure is `Structure` with fields `[("cbData", DWORD), ("pbData", POINTER(c_char))]`. `CryptProtectData` and `CryptUnprotectData` signatures require `dwFlags=0` for user-scope (no `CRYPTPROTECT_LOCAL_MACHINE`). `LocalFree` must be called on the output blob's `pbData` pointer to avoid memory leaks. On non-Windows (CI cross-platform tests), the encrypt/decrypt functions must be no-ops or use a mock layer â€” do not import `ctypes.windll` at module level.

The `cleanup()` function currently checks `state.firewall_rules_created` before removing firewall rules (D-6 removes this flag). After this task, cleanup should call `remove_firewall_rules()` unconditionally. Verify that change is made here if task 7 hasn't landed yet (coordinate with wave sequencing).
