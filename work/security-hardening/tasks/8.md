# Task 8: Harden CSP modification — minimal nonce injection

## Required Skills
Before starting, load: /skill:code-writing

## Description
The current `_modify_csp()` in `proxy_addon.py` has two security weaknesses. First, when no `script-src` directive exists, it creates one by copying all tokens from `default-src` verbatim — this can promote `unsafe-inline`, `unsafe-eval`, and `unsafe-hashes` into the new `script-src`, weakening a site's XSS protection. Second, `content-security-policy-report-only` is iterated in the same loop as the enforcing header and modified, which can leak information about site policy or silence legitimate violation reports. Both issues must be fixed with minimal, targeted changes to `_modify_csp` and the response header loop.

## What to do

1. Update `_modify_csp(csp_value: str, nonce: str) -> str` in `proxy_addon.py`:
   - When deriving a new `script-src` from `default-src` (the `not has_script_src` branch), filter out these tokens from the copied source list: `'unsafe-inline'`, `'unsafe-eval'`, `'unsafe-hashes'`. Match case-insensitively.
   - Keep all other `default-src` tokens unchanged (e.g., `'self'`, `https:`, domain allowlists, existing nonces/hashes).
   - Append the injection nonce as the final token: `script-src <filtered-sources> 'nonce-{nonce}'`.
   - When `script-src` already exists (the `has_script_src = True` branch), the current behavior (append nonce only) is correct — do not change it.
   - When neither `script-src` nor `default-src` exists, the current behavior (minimal `script-src 'nonce-{nonce}'`) is correct — do not change it.

2. Update the response header loop in `GeoFixAddon.response()`:
   - Change the loop `for header_name in ("content-security-policy", "content-security-policy-report-only"):` to only process `"content-security-policy"`.
   - Do not modify `content-security-policy-report-only` at all.

3. Do not change any other behavior of `_modify_csp` or `GeoFixAddon`.

## Details
The dangerous code path is in `_modify_csp` lines 104–115 (current file):

```python
if not has_script_src:
    for directive in directives:
        parts = directive.split()
        if parts and parts[0].lower() == "default-src":
            new_directives.append(f"script-src {' '.join(parts[1:])} {nonce_value}")
```

`' '.join(parts[1:])` copies every token including `'unsafe-inline'`. The fix filters `parts[1:]` before joining. The filter set is: `{"'unsafe-inline'", "'unsafe-eval'", "'unsafe-hashes'"}` — compare lowercased.

The report-only issue is in `GeoFixAddon.response()` line 193: the loop iterates both header names. Removing `"content-security-policy-report-only"` from the tuple (or changing to a single string lookup) is the complete fix.

The `'unsafe-hashes'` directive is less common but must be filtered for consistency — a `default-src 'unsafe-hashes'` policy would otherwise promote hash-based inline execution to script-src.
