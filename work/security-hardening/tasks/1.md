# Task 1: Per-session CA with ephemeral confdir and thumbprint-based removal

## Required Skills
Before starting, load: /skill:code-writing

## Description
Currently, geo-fix installs the mitmproxy CA cert from `~/.mitmproxy/` once and never removes it, leaving the private key permanently accessible to any process running as the current user. This task replaces that pattern with a per-session isolated `confdir`: a fresh `tempfile.mkdtemp()` is created at startup with a restricted Windows ACL, and mitmproxy is pointed at it so it generates a new CA each session. After mitmproxy starts, the CA thumbprint is captured via `certutil` and stored in `ProxyState` so cleanup can use `certutil -delstore -user Root <thumbprint>` to remove precisely the right cert. The session tmpdir (containing the CA private key) is deleted entirely on cleanup.

## What to do

1. **Add fields to `ProxyState`** in `src/system_config.py`:
   - `session_tmpdir: Optional[str] = None` — path to the per-session temp directory
   - `ca_thumbprint: Optional[str] = None` — SHA-1 thumbprint recorded at install time
   - `session_id: Optional[str] = None` — uuid4 string generated at startup

2. **Add `create_session_tmpdir()` function** in `src/system_config.py`:
   - Call `tempfile.mkdtemp(prefix="geo-fix-")` to create the directory
   - On Windows, run `icacls <dir> /inheritance:r /grant:r %USERNAME%:(OI)(CI)F` to restrict ACL
   - Return the path as a string

3. **Modify `install_ca_cert(confdir: str) -> Optional[str]`** in `src/system_config.py`:
   - Accept `confdir` parameter instead of using the global `MITMPROXY_CA_DIR`
   - Locate the cert at `Path(confdir) / "mitmproxy-ca-cert.pem"`
   - After successful `certutil -addstore`, extract the thumbprint by parsing `certutil -store -user Root` output for the cert subject "mitmproxy"
   - Return the thumbprint string on success, `None` on failure
   - Remove the now-unnecessary `_restrict_ca_key_permissions()` call (ACL is on the whole tmpdir)

4. **Modify `uninstall_ca_cert(thumbprint: Optional[str] = None)`** in `src/system_config.py`:
   - If `thumbprint` is provided, run `certutil -delstore -user Root <thumbprint>`
   - Fall back to the current name-based removal (`certutil -delstore -user Root mitmproxy`) if thumbprint is `None`

5. **Add `delete_session_tmpdir(session_tmpdir: Optional[str])`** in `src/system_config.py`:
   - If path is set and directory exists, call `shutil.rmtree(session_tmpdir, ignore_errors=True)`
   - Log the deletion

6. **Update `cleanup()` in `src/system_config.py`**:
   - Call `uninstall_ca_cert(state.ca_thumbprint)` BEFORE `delete_state()`
   - Call `delete_session_tmpdir(state.session_tmpdir)` BEFORE `delete_state()`

7. **Update `main()` in `src/main.py`**:
   - Generate `session_id = str(uuid.uuid4())` at startup (add `import uuid`)
   - Call `create_session_tmpdir()` and store the result
   - Pass `confdir=session_tmpdir` to mitmproxy's `Options` in `_start_mitmproxy()`
   - After mitmproxy is confirmed running, call `install_ca_cert(session_tmpdir)` and store returned thumbprint
   - Include `session_id`, `session_tmpdir`, and `ca_thumbprint` in the `ProxyState` constructor

8. **Remove global `MITMPROXY_CA_DIR` / `MITMPROXY_CA_CERT` constants** from `src/system_config.py` (they are no longer used by the main code path; keep only if still referenced by tests).

## Details

**Current state of `install_ca_cert()`** (lines 254–278 of `system_config.py`): takes no parameters, reads from `MITMPROXY_CA_CERT = Path.home() / ".mitmproxy" / "mitmproxy-ca-cert.pem"`, runs `certutil -addstore`, then calls `_restrict_ca_key_permissions()` to chmod the key file. Does not capture or return a thumbprint.

**Current state of `uninstall_ca_cert()`** (lines 281–293): removes by name literal `"mitmproxy"` — `certutil -delstore -user Root mitmproxy`. This is fragile: it could miss the cert if renamed or remove the wrong cert.

**Current state of `ProxyState`** (lines 42–52): does not have `session_id`, `ca_thumbprint`, or `session_tmpdir` fields. `from_json()` enforces strict schema validation (line 62–65) — unknown fields raise `ValueError`. New fields must be added to the dataclass to pass through serialization.

**Current state of `cleanup()`** (lines 374–403): restores proxy, restores Firefox, conditionally removes firewall rules, then deletes state file. `uninstall_ca_cert()` is **never called** — this is the root cause of AC-2.1 failure.

**Current state of `main()`** (lines 239–261): calls `set_wininet_proxy()` and `set_firefox_proxy()` before `_start_mitmproxy()`. This is the wrong order (fixed in Task 5). The `_start_mitmproxy()` passes `listen_port=PROXY_PORT` (hardcoded 8080) and no `confdir`.

**Thumbprint extraction**: `certutil -store -user Root` outputs one block per cert. Each block contains a `Cert Hash(sha1):` line. After installing, re-query the store and match the cert by subject containing "mitmproxy" to extract its thumbprint. Alternatively, use `certutil -dump <cert_file>` which outputs `Cert Hash(sha1): <hex>` directly — this is simpler and more reliable.
