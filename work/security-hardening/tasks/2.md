# Task 2: Add CA removal and tmpdir cleanup to all exit paths

## Required Skills
Before starting, load: /skill:code-writing

## Description
Task 1 adds `uninstall_ca_cert()` and `delete_session_tmpdir()` to `cleanup()`, but that only covers the case where cleanup is called with a valid state. This task audits every termination path in the codebase and ensures the CA cert and session tmpdir are removed in all of them: the `--stop` signal path, the `--cleanup` recovery path, SIGTERM handler, atexit handler, and the watchdog recovery path (which is set up in Task 4). The key invariant is that `uninstall_ca_cert` and `delete_session_tmpdir` must be called before `delete_state()` so that thumbprint and path information is available when needed.

## What to do

1. **Verify `cleanup()` in `src/system_config.py`** calls `uninstall_ca_cert(state.ca_thumbprint)` and `delete_session_tmpdir(state.session_tmpdir)` in the right order (before `delete_state()`). This should already be done by Task 1 — confirm and do not regress.

2. **Audit `_do_cleanup()` in `src/main.py`** (lines 130–145):
   - Currently calls `cleanup(state)` via `load_state()`. Since `cleanup()` now calls cert removal, this path is covered — but only if `load_state()` returns a valid state.
   - Ensure the function does NOT catch and silently swallow exceptions from `uninstall_ca_cert` or `delete_session_tmpdir`.

3. **Audit `_handle_stop()` in `src/main.py`** (lines 93–116):
   - Sends SIGTERM to the running process; if cleanup doesn't happen within ~2 seconds, calls `cleanup(state)` directly.
   - Confirm this calls the updated `cleanup()` that includes cert removal and tmpdir deletion.

4. **Audit `_handle_cleanup()` in `src/main.py`** (lines 119–127):
   - Called when user runs `geo-fix --cleanup`. Calls `cleanup(state)`.
   - Confirm `state` loaded here includes `ca_thumbprint` and `session_tmpdir` (these fields are now in `ProxyState` from Task 1).

5. **Verify SIGTERM handler** (line 222 in `src/main.py`):
   - `signal.signal(signal.SIGTERM, lambda sig, frame: (_do_cleanup(), sys.exit(0)))` — this calls `_do_cleanup()` which calls `cleanup()`. Confirm chain is intact.

6. **Verify atexit handler** (line 221):
   - `atexit.register(_do_cleanup)` — same chain. Confirm intact.

7. **Add a `--cleanup` integration test** that verifies cert removal by thumbprint:
   - Write `test/test_cleanup_paths.py::test_cleanup_removes_cert` — install a test cert, write a state file with the thumbprint, call `cleanup()`, assert cert is gone from store.
   - Write `test/test_cleanup_paths.py::test_cleanup_deletes_tmpdir` — create a real tmpdir, write state with `session_tmpdir` path, call `cleanup()`, assert directory is gone.
   - Write `test/test_cleanup_paths.py::test_cleanup_order_cert_before_state` — mock `delete_state` and verify `uninstall_ca_cert` is called first (use `unittest.mock.call_args_list` ordering).

8. **Document the cleanup contract** with a short module-level comment in `system_config.py`'s `cleanup()` function: cleanup order must be: proxy → firefox → firewall → cert → tmpdir → state file.

## Details

**Current `cleanup()` order** (lines 374–403 of `system_config.py`):
1. `unset_wininet_proxy(original)` — restores registry
2. `unset_firefox_proxy(state.firefox_prefs_backup)` — restores Firefox
3. `remove_firewall_rules()` — conditional on `state.firewall_rules_created`
4. `delete_state()` — removes state file

Missing steps: `uninstall_ca_cert()` and `delete_session_tmpdir()` are not called at all. Task 1 adds them. This task verifies the complete chain and all entry points.

**`_do_cleanup()` guard** (lines 130–145): uses `_cleanup_done` boolean and `_cleanup_lock` to ensure idempotency. The atexit and SIGTERM handler both route through here. The guard is correct; just ensure the inner `cleanup(state)` call propagates new behavior from Task 1.

**State schema validation**: `ProxyState.from_json()` rejects unknown fields (strict schema). Since Task 1 adds `session_id`, `session_tmpdir`, `ca_thumbprint` as Optional fields with default `None`, old state files from before Task 1 will fail to load (unknown fields). This is acceptable behavior — old state without these fields cannot do thumbprint-based removal anyway; it will fall back to name-based removal via the `None` path.

**Firewall conditional** (line 398–399): `remove_firewall_rules()` is only called if `state.firewall_rules_created`. Per tech-spec D-6, this flag will be removed in Task 7, making firewall removal unconditional. Do not pre-empt Task 7 here.
