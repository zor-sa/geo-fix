# User Spec: security-hardening — Устранение угроз безопасности geo-fix

## Что делаем

Полное устранение 12 выявленных угроз безопасности в geo-fix: от критических (перманентный CA-сертификат, утечка приватного ключа) до низких (молчаливый пропуск шагов wizard). Каждая угроза должна быть закрыта наилучшим доступным решением без полумер.

## Зачем

geo-fix устанавливает корневой CA-сертификат и модифицирует системные настройки прокси. Текущая реализация оставляет CA-сертификат навсегда после первого запуска, не восстанавливает прокси при аварийном завершении, и содержит ряд других уязвимостей. Любой процесс пользователя может прочитать приватный ключ CA и выполнить MITM всего HTTPS-трафика. Без исправлений инструмент представляет бо́льшую угрозу безопасности, чем проблема которую решает.

## Контекст

- Анализ угроз выполнен, все 12 уязвимостей задокументированы
- Код полностью написан, нужен security hardening существующей реализации
- Целевая ОС: Windows 10/11
- Архитектура (mitmproxy + JS-инъекция) не меняется

## Угрозы и требования

### CRITICAL

#### T-1: CA-сертификат и приватный ключ

**Угроза:** CA-сертификат mitmproxy устанавливается в CurrentUser Root store навсегда. Приватный ключ лежит в `~/.mitmproxy/` и доступен любому процессу текущего пользователя.

**Требование:** Генерировать свежий CA-сертификат при каждом запуске прокси. При остановке — удалить сертификат из store и уничтожить приватный ключ. Между сессиями ничего не должно оставаться. Ключ во время работы должен быть защищён максимально (owner-read-only, выделенная директория).

**Критерий приёмки:**
- AC-1.1: После `--stop` — CA-сертификат отсутствует в `certmgr.msc` → CurrentUser → Root
- AC-1.2: После `--stop` — файл приватного ключа CA не существует на диске
- AC-1.3: Каждый запуск генерирует новый CA (отпечатки разные)
- AC-1.4: Во время работы приватный ключ доступен только текущему пользователю (icacls verify)

#### T-2: cleanup() не удаляет CA-сертификат

**Угроза:** `uninstall_ca_cert()` не вызывается ни из `cleanup()`, ни из других путей завершения.

**Требование:** `uninstall_ca_cert()` вызывается из КАЖДОГО пути завершения: `--stop`, `--cleanup`, SIGTERM, atexit, аварийное восстановление.

**Критерий приёмки:**
- AC-2.1: `uninstall_ca_cert()` присутствует в `cleanup()` и вызывается перед удалением state file
- AC-2.2: При `--cleanup` (восстановление после краша) — CA-сертификат удаляется

### HIGH

#### T-3: Прокси в реестре не восстанавливается при hard kill

**Угроза:** При `taskkill /F` или BSOD прокси в реестре остаётся включённым, весь HTTP-трафик браузеров ломается.

**Требование:** Механизм, переживающий принудительное завершение: watchdog-процесс, Windows Job Object, или scheduled task при старте, которая выполняется при следующей загрузке. Прокси должен быть восстановлен в течение 5 секунд или при следующем boot.

**Критерий приёмки:**
- AC-3.1: После `taskkill /F /PID <geo-fix>` — ProxyEnable=0 в реестре в течение 5 секунд
- AC-3.2: Watchdog/recovery процесс не требует ручного запуска
- AC-3.3: После reboot — если geo-fix не запущен, прокси гарантированно выключен

#### T-4: Firefox user.js — rename вместо copy, утечка enterprise_roots

**Угроза:** `rename()` теряет оригинал при краше между rename и записью нового файла. `security.enterprise_roots.enabled=true` не откатывается.

**Требование:** Использовать copy для backup. Явно откатывать `enterprise_roots` при cleanup.

**Критерий приёмки:**
- AC-4.1: Оригинальный user.js сохраняется через `shutil.copy2()`, не через `rename()`
- AC-4.2: После `--stop` — `security.enterprise_roots.enabled` отсутствует в user.js (если не было изначально)
- AC-4.3: При краше между backup и записью — оригинальный user.js остаётся на месте

#### T-5: install.bat без проверки хэшей

**Угроза:** Python и get-pip.py скачиваются без SHA-256 верификации. MITM или DNS poisoning → произвольный код.

**Требование:** SHA-256 хэши захардкожены в скрипте. Каждый файл проверяется после скачивания. При несовпадении — exit с ошибкой и удаление файла.

**Критерий приёмки:**
- AC-5.1: SHA-256 хэш Python embed zip проверяется после download
- AC-5.2: SHA-256 хэш get-pip.py проверяется после download
- AC-5.3: При hash mismatch — скрипт завершается с ненулевым кодом и сообщением об ошибке
- AC-5.4: Скачанный файл с невалидным хэшем удаляется

### MEDIUM

#### T-6: Порт 8080 захардкожен, прокси в реестре до старта mitmproxy

**Угроза:** Если 8080 занят — прокси в реестре указывает в никуда. Реестр пишется до запуска mitmproxy.

**Требование:** Порт конфигурируемый или auto-select. Реестр модифицируется ТОЛЬКО после успешного bind порта mitmproxy.

**Критерий приёмки:**
- AC-6.1: `--port <N>` или автоматический выбор свободного порта
- AC-6.2: WinINET proxy устанавливается только после подтверждения что mitmproxy слушает
- AC-6.3: Если порт занят — понятное сообщение об ошибке, реестр не тронут

#### T-7: JS-инъекция ослабляет CSP

**Угроза:** Модификация CSP может ослабить защиту сайтов от XSS.

**Требование:** Минимальная модификация — только добавление nonce в script-src. Не копировать unsafe-* из default-src. Не модифицировать report-only.

**Критерий приёмки:**
- AC-7.1: Если в default-src есть `unsafe-eval`/`unsafe-inline` — они НЕ копируются в создаваемый script-src
- AC-7.2: `content-security-policy-report-only` не модифицируется
- AC-7.3: Существующие директивы CSP кроме script-src не затрагиваются

#### T-8: Firewall-правила с неверными путями, не удаляются

**Угроза:** Пути к браузерам невалидные (`C:\chrome.exe`). Правила создаются, но не блокируют ничего.

**Требование:** Автоматическое определение путей из реестра/файловой системы. Удаление правил при stop.

**Критерий приёмки:**
- AC-8.1: Путь Chrome определяется из `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\chrome.exe` или поиском в Program Files
- AC-8.2: Аналогично для Edge и Firefox
- AC-8.3: Все созданные правила удаляются при `--stop`
- AC-8.4: Если путь к браузеру не найден — правило для него не создаётся, логируется предупреждение

#### T-9: State file без защиты целостности

**Угроза:** State file можно подменить для инъекции вредоносного proxy_server при restore.

**Требование:** HMAC-SHA256 с машинно-локальным секретом. При несовпадении — отказ от state file, предупреждение, восстановление чистого состояния.

**Критерий приёмки:**
- AC-9.1: State file содержит HMAC-SHA256 подпись
- AC-9.2: При загрузке HMAC проверяется до применения каких-либо значений
- AC-9.3: Подменённый state file отклоняется с логом warning
- AC-9.4: После отклонения — применяется чистое состояние (прокси выключен, настройки по умолчанию)

### LOW

#### T-10: Skip wizard молча ломает HTTPS

**Угроза:** Кнопка "Пропустить" помечает setup complete без установки CA. HTTPS перехват не работает.

**Требование:** Явное предупреждение с описанием последствий. Пользователь должен подтвердить.

**Критерий приёмки:**
- AC-10.1: При нажатии "Пропустить" — диалог с предупреждением о неработоспособности HTTPS
- AC-10.2: Пользователь должен явно подтвердить пропуск

#### T-11: Console wizard пропускает firewall

**Угроза:** Console wizard не предлагает настройку firewall.

**Требование:** Console wizard предлагает те же шаги что и GUI.

**Критерий приёмки:**
- AC-11.1: Console wizard спрашивает о firewall-правилах
- AC-11.2: Console wizard показывает те же инструкции по DNS что и GUI

#### T-12: Launcher .bat файлы доступны на запись

**Угроза:** Малварь может заменить .bat файл для persistence.

**Требование:** Read-only permissions при установке или integrity check.

**Критерий приёмки:**
- AC-12.1: .bat файлы создаются с атрибутом read-only
- AC-12.2: Или: при запуске проверяется целостность launcher файла

## Тестирование

- Unit-тесты: каждый security fix покрыт
- Integration-тесты: реальные операции с реестром, cert store, firewall на Windows
- E2E-тест: hard kill процесса + верификация полной очистки в течение 5 секунд
- install.bat: тест hash mismatch → exit code != 0
- State file: тест HMAC tampering → rejection + clean state

## Вне скоупа

- Изменение архитектуры (mitmproxy + JS-инъекция остаётся)
- Аутентификация пользователей
- Сетевое шифрование сверх того что даёт mitmproxy
- Поддержка других ОС
