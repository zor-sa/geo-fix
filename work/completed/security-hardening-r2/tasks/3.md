## Description

This task addresses two security/reliability issues found in analyst review: R-5 (VPN drop during operation goes unnoticed) and W-1 (watchdog can die without main process noticing).

**R-5 ŌĆö VPN monitoring:** Currently, geo-fix only checks VPN status at startup. If the VPN disconnects mid-session, the user continues using a proxy that spoofs geo signals but exposes their real IP ŌĆö which silently breaks the security model. A background monitor must detect this and alert the user immediately via stderr print and (when tray notification is added) tray toast.

**W-1 ŌĆö Watchdog health:** The watchdog subprocess is spawned once at startup but is never checked again. If it dies (crash, OOM, etc.), the main process loses its safety net ŌĆö if the main process then dies mid-session without triggering `_do_cleanup()`, system proxy settings and CA cert remain installed with no automatic recovery. The monitor must detect watchdog death and respawn it.

**Current state:** `_monitor_loop()` already exists in `main.py` and already includes both VPN checking and watchdog respawn logic ŌĆö it was implemented partially in a previous session. The code is present but the tests are missing. This task's job is to verify the implementation is correct and complete, then add the required tests.

The monitor thread runs every 60 seconds using `stop_event.wait(timeout=60)`, so it exits cleanly when geo-fix shuts down.
