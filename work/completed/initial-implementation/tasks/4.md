# Task 4: Windows system configuration

## Description

Create the module that manages Windows proxy settings (WinINET registry HKCU for Chrome/Edge + Firefox prefs.js), CA certificate installation (certutil to CurrentUser store for Chrome/Edge + Firefox enterprise_roots or NSS), optional firewall rules (netsh for WebRTC STUN blocking), and atomic state file for crash recovery. All changes must be reversible.

## Context Files
- `src/presets.py` — presets data
- `work/geo-fix/tech-spec.md` — D2 Firefox support, D3 firewall, D9 state file, D10 proxy binding

## Acceptance Criteria
- [ ] WinINET proxy set/unset via registry HKCU (no admin needed)
- [ ] Firefox proxy configured via prefs.js (network.proxy.* settings)
- [ ] CA cert installed to CurrentUser store via certutil (no admin)
- [ ] Firefox CA trust enabled via security.enterprise_roots.enabled=true
- [ ] Optional firewall rules created via netsh (admin required, per-app, unique naming geo-fix-*)
- [ ] State file written atomically (write temp, rename)
- [ ] State file stores: original proxy settings, active preset name, PID, timestamp
- [ ] State file validated against strict schema on read
- [ ] All changes reversible: cleanup function restores original state
- [ ] Cleanup works even if partially applied (e.g., proxy set but cert failed)
- [ ] CA key file permissions restricted to current user after generation

## TDD Anchor
- Test: mock winreg, verify correct registry keys set and reverted
- Test: mock subprocess (certutil), verify correct arguments for install/uninstall
- Test: mock subprocess (netsh), verify per-app rules with geo-fix-* naming
- Test: state file atomic write (temp + rename pattern)
- Test: state file validation rejects unknown fields
- Test: cleanup after partial setup (proxy set, cert failed)
