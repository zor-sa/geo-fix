# Task 1: Country presets module

## Description

Create the presets data module with all 4 country configurations (timezone, coordinates, language, Accept-Language header). Each preset must have internally consistent data that matches what a real user in that country would produce. Include target domain list for JS injection scope.

## Context Files
- `work/geo-fix/user-spec.md` — FR-2 preset data
- `work/geo-fix/tech-spec.md` — D8 target domains

## Acceptance Criteria
- [ ] PRESETS dict contains entries for US, DE, NL, GB
- [ ] Each preset has: timezone (IANA), coordinates (lat/lon), language (BCP47), accept_language header string
- [ ] Timezone offsets are correct for both DST and standard time
- [ ] Coordinates are within the correct country boundaries
- [ ] TARGET_DOMAINS list contains *.google.com, *.googleapis.com, *.gstatic.com
- [ ] All preset data is internally consistent (e.g., US timezone matches US coordinates)

- Test DST offset computation for summer and winter dates for each preset
- Test TARGET_DOMAINS contains expected entries
