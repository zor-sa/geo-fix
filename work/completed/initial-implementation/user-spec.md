# User Spec: geo-fix — Подмена геосигналов для обхода геоблокировок Google

## Что делаем

Windows-приложение, которое перехватывает и подменяет сигналы геолокации на лету — без изменения системных настроек — чтобы в связке с VPN обеспечить доступ к гео-заблокированным сервисам Google (NotebookLM, Gemini).

## Зачем

Google определяет геолокацию не только по IP, но и по timezone, WebRTC, DNS, языку браузера и Geolocation API. VPN меняет только IP — остальные сигналы выдают реальное местоположение. Нужен инструмент, который подменит все сигналы консистентно, чтобы Google не видел несоответствий.

## Контекст

- VPN уже установлен и работает (меняет IP)
- Целевые сервисы: notebooklm.google.com, gemini.google.com
- Целевая ОС: Windows 10/11
- Пользователь: нетехнический, нужен простой запуск

## Функциональные требования

### FR-1: Подмена 5 геосигналов

| Сигнал | Что подменяется | Метод |
|---|---|---|
| Timezone | `Intl.DateTimeFormat`, `Date.getTimezoneOffset()`, `Temporal.Now.timeZoneId` | Перехват через прокси (JS-инъекция) |
| WebRTC | Утечка реального IP через STUN-серверы | Перехват `RTCPeerConnection` через JS-инъекцию |
| DNS | DNS-запросы мимо VPN-туннеля | Перенаправление DNS через зашифрованный канал |
| Язык | `Accept-Language` заголовок, `navigator.language`, `navigator.languages` | Подмена заголовка через прокси + JS-инъекция |
| Geolocation API | GPS/WiFi координаты через `navigator.geolocation` | Подмена через JS-инъекцию |

### FR-2: Пресеты стран

Минимум 4 пресета с консистентным набором данных:

| Страна | Timezone | Координаты | Язык |
|---|---|---|---|
| US | America/New_York | Washington DC (38.90, -77.04) | en-US |
| DE | Europe/Berlin | Berlin (52.52, 13.41) | de-DE |
| NL | Europe/Amsterdam | Amsterdam (52.37, 4.90) | nl-NL |
| GB | Europe/London | London (51.51, -0.13) | en-GB |

### FR-3: CLI-интерфейс

```
geo-fix.exe US        # включить маскировку для США
geo-fix.exe DE        # переключить на Германию
geo-fix.exe --stop    # выключить, вернуть всё как было
```

### FR-4: Иконка в системном трее

- Отображается при активной маскировке
- Показывает текущую страну (флаг или аббревиатура)
- Правая кнопка → контекстное меню:
  - `Активна: США (US)` — статус (неактивный пункт)
  - Разделитель
  - `Переключить страну →` подменю: US, DE, NL, GB
  - `Выключить` — корректное завершение

### FR-5: Чистое включение/выключение

- При запуске: включить прокси, начать подмену
- При `--stop` или "Выключить" из трея: снять прокси, вернуть все настройки
- Никаких постоянных изменений в системе после выключения

### FR-6: Ярлыки на рабочем столе

- `Включить (US)` — запуск с параметром US
- `Включить (DE)` — запуск с параметром DE
- `Выключить` — запуск с параметром --stop

## Нефункциональные требования

### NFR-1: Без прав администратора
Приложение должно работать без прав администратора (насколько технически возможно). Допустимо: первоначальная установка CA-сертификата может потребовать подтверждения.

### NFR-2: Бесплатное, без платных зависимостей
Все компоненты open-source или бесплатные.

### NFR-3: Системная работа
Работает для любого браузера (Chrome, Edge, Firefox) и десктопных приложений, использующих системный прокси.

### NFR-4: Минимальное влияние на производительность
Прокси не должен заметно замедлять интернет при обычном использовании.

## Критерии приёмки

### AC-1: Отсутствие утечек на fingerprint-тестах
На https://abrahamjuliot.github.io/creepjs/ — нет красных флагов, нет упоминания Russia. Timezone, язык, геолокация соответствуют выбранной стране.

### AC-2: WebRTC не утекает
На https://browserleaks.com/webrtc — не показывается реальный IP.

### AC-3: DNS не утекает
На https://dnsleaktest.com — не показывается российский провайдер/DNS.

### AC-4: IP соответствует стране
На https://ipapi.co — IP и страна соответствуют VPN-стране.

### AC-5: Доступ к Google-сервисам
notebooklm.google.com и gemini.google.com открываются и работают.

### AC-6: Чистая деактивация
После `--stop` все системные настройки возвращены в исходное состояние. Проверка: прокси выключен, DNS не изменён.

### AC-7: Трей-иконка функциональна
Иконка появляется, контекстное меню работает, переключение страны и выключение работают.

## Ограничения и допущения

- VPN уже настроен и работает — приложение не управляет VPN
- CA-сертификат для HTTPS-прокси нужно установить один раз при первом запуске
- Приложения с certificate pinning могут не работать через прокси
- WebRTC-блокировка через JS (основной метод, без админ-прав) менее надёжна, чем через firewall (опциональный метод, требует админ-прав)
- Firefox требует отдельной настройки прокси и CA-сертификата (автоматизируется setup wizard)

## Стратегия тестирования

- **Smoke-тесты**: автоматическая проверка что прокси запустился, сертификат установлен, JS-инъекция работает
- **E2E-тесты**: проверка на creepjs, browserleaks, dnsleaktest через headless browser
- **Ручная проверка**: доступ к NotebookLM и Gemini
